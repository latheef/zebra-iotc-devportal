# dataEVT

Tag and barcode read data events published by the reader during an active RFID inventory operation. Each event contains tag data and optionally barcode data with EPC, RSSI, antenna, timestamp, and access results.

## Topic

| | Topic |
|---|---|
| **data1event** | `{tenantId}/DATA/clients/data1event/{deviceSerial}` |
| **data2event** | `{tenantId}/DATA/clients/data2event/{deviceSerial}` |

## Direction

Reader → Application

## Payload

### RFID Tag Data

```json
{
  "type": "BALANCED_PERFORMANCE",
  "timestamp": "2019-08-24T14:15:22Z",
  "data": {
    "tagData": [
      {
        "EPCid": "BEDD11112222333344445555",
        "EPC": "BEDD11112222333344445555",
        "TID": "E2003412013BFD000B4E16D21D030143000D5FFBFFFFDC60",
        "USER": "12343333123456781234567812345678",
        "channel": 911.75,
        "eventNum": 1,
        "format": "epc",
        "peakRssi": -39,
        "phase": 0,
        "seenCount": 1,
        "accessResults": [
          "READ-EPC-SUCCESS",
          "READ-TID-SUCCESS",
          "WRITE-USER-No Response from Tag"
        ]
      }
    ]
  }
}
```

### With Barcode Data

```json
{
  "type": "FAST_READ",
  "timestamp": "2026-05-14T10:32:16.456Z",
  "data": {
    "tagData": [
      {
        "EPCid": "30340242A02C00000000001A",
        "EPC": "30340242A02C00000000001A",
        "eventNum": 2,
        "format": "epc",
        "peakRssi": -48,
        "seenCount": 3,
        "firstSeenTime": 1747218136100,
        "lastSeenTime": 1747218136456
      }
    ],
    "barcodeData": [
      {
        "symbology": "CODE_39",
        "decodedBarcode": "ITEM-00123456"
      }
    ]
  }
}
```

| Field | Type | Description |
|---|---|---|
| `type` | string | Operating mode that produced the data (see enum below) |
| `timestamp` | string | Event publish timestamp in ISO 8601 format |
| `data` | object | Tag and barcode data container |
| `data.tagData` | array | Array of tag read records |
| `data.barcodeData` | array | Array of barcode scan records (present when barcodes are scanned) |

### `type` Enum

| Value | Description |
|---|---|
| `FAST_READ` | High-speed read mode |
| `CYCLE_COUNT` | Cycle count inventory mode |
| `DENSE_READERS` | Dense reader environment mode |
| `OPTIMAL_BATTERY` | Battery-optimized mode |
| `BALANCED_PERFORMANCE` | Balanced speed/battery mode |
| `ADVANCED` | Advanced custom configuration mode |

### Tag Data Fields (`data.tagData[]`)

| Field | Type | Description |
|---|---|---|
| `eventNum` | number | Event sequence number (required) |
| `format` | string | EPC encoding format, typically `"epc"` (required) |
| `EPCid` | string | EPC identifier — unique identifier encoded on the tag (hex string) |
| `EPC` | string | EPC memory bank content (hex string) |
| `TID` | string | Tag Identifier memory bank — manufacturer-assigned, immutable (hex string) |
| `USER` | string | User memory bank — application-defined data (hex string) |
| `RESERVED` | string | Reserved memory bank — kill and access passwords (hex string) |
| `PC` | string | PC bits of the inventoried tag (hex string) |
| `CRC` | string | CRC bits of the inventoried tag (hex string) |
| `XPC` | string | XPC bits, if present (hex string) |
| `channel` | number | RF channel in MHz when the tag was read (only when `reportFilter` duration is `0`) |
| `peakRssi` | number | Signal strength in dBm (peak value since last report) |
| `phase` | number | RF phase angle in degrees (only when `reportFilter` duration is `0`) |
| `seenCount` | number | Times the tag was inventoried since the previous report (always `1` when `reportFilter` duration is `0`) |
| `firstSeenTime` | number | Timestamp in milliseconds since epoch when the tag was first seen |
| `lastSeenTime` | number | Timestamp in milliseconds since epoch when the tag was last seen |
| `accessResults` | array | Results of access operations (read/write/lock/kill) as string array |
| `userDefined` | string | Custom string value configured in operating mode |
| `MAC` | string | MAC address of the reader that inventoried the tag |
| `HOSTNAME` | string | Hostname of the reader that inventoried the tag |

### Barcode Data Fields (`data.barcodeData[]`)

| Field | Type | Description |
|---|---|---|
| `symbology` | string | Barcode symbology: `CODE_39` |
| `decodedBarcode` | string | Decoded barcode string value (required) |

## Notes

- The `data1event` and `data2event` topics carry the same event type. Two topics are provided for different QoS or subscriber routing in high-throughput scenarios.
- The fields present in each tag record depend on the `tagReportConfig` configured via `set_operating_mode`. Only enabled fields appear in the payload.
- `EPCid` is typically the same as `EPC` — both represent the Electronic Product Code.
- `peakRssi` values are negative dBm. Higher values (closer to 0) indicate stronger signal.
- `firstSeenTime` and `lastSeenTime` are milliseconds since Unix epoch (not ISO 8601 strings).
- `channel` and `phase` are only included when `reportFilter` duration is set to `0` (per-read reporting).
- High tag populations can generate significant data volume. Use `seenCountThreshold` and post-filters to manage throughput.
