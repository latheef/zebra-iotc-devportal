# dataEVT

Tag read data events published by the reader during an active RFID operation. Each event contains one or more tag reads with EPC, RSSI, antenna, timestamp, and optional fields based on the `tagReportConfig` in the operating mode.

## Topic

| | Topic |
|---|---|
| **data1event** | `{tenantId}/DATA/clients/data1event/{deviceSerial}` |
| **data2event** | `{tenantId}/DATA/clients/data2event/{deviceSerial}` |

## Direction

Reader → Application

| Field | Type | Description |
|---|---|---|
| `event` | string | Always `"dataEVT"` |
| `deviceSerial` | string | Serial number of the reporting device |
| `timestamp` | string | Event publish timestamp in ISO 8601 format |
| `tags` | array | Array of tag read records |
| `tags[].epc` | string | Electronic Product Code in hexadecimal |
| `tags[].rssi` | number | Received Signal Strength Indicator in dBm |
| `tags[].antennaId` | integer | Antenna that read the tag (present when `includeAntennaId` is `true`) |
| `tags[].firstSeenTime` | string | First read timestamp (present when `includeFirstSeenTime` is `true`) |
| `tags[].lastSeenTime` | string | Last read timestamp (present when `includeLastSeenTime` is `true`) |
| `tags[].phase` | number | RF phase angle in degrees (present when `includePhase` is `true`) |
| `tags[].channelIndex` | integer | RF channel index used for the read (present when `includeChannel` is `true`) |
| `tags[].tid` | string | Tag identifier memory bank in hexadecimal (present when `includeTid` is `true`) |
| `tags[].userMemory` | string | User memory bank in hexadecimal (present when `includeUserMemory` is `true`) |
| `tags[].seenCount` | integer | Number of times the tag was read in the reporting interval |

## Notes

- The `data1event` and `data2event` topics carry the same event type. Two topics are provided to allow different QoS or subscriber routing for high-throughput scenarios.
- The fields present in each tag record depend on the `tagReportConfig` configured via `set_operating_mode`. Only enabled fields appear in the payload.
- `epc` is always present in every tag record regardless of configuration.
- `rssi` values are negative dBm. Higher values (closer to 0) indicate stronger signal and closer proximity.
- `phase` values range from 0 to 360 degrees and can be used for motion detection or distance estimation algorithms.
- Tag events are batched; a single `dataEVT` may contain multiple tags. The batch size depends on the read rate and internal buffering.
- High tag populations and dense read environments can generate significant data volume. Use `seenCountThreshold` and post-filters to manage throughput.
