# get_eth

Queries the Ethernet interface status, link state, and IPv4 configuration of the device.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

```json
{
  "command": "get_eth",
  "requestId": "4"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_eth"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Response Payload

```json
{
  "command": "get_eth",
  "requestId": "4",
  "statusCode": 0,
  "statusMessage": "Success",
  "interfaceDetails": {
    "status": "UP"
  },
  "linkStatus": {
    "status": "LINKED",
    "linkSpeed": "100Mbps"
  },
  "ipv4Configuration": {
    "ipAddress": "192.168.1.50",
    "subnetMask": "255.255.255.0",
    "enableDhcp": true,
    "gateway": "192.168.1.1",
    "dns1": "8.8.8.8",
    "dns2": "8.8.4.4"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `statusCode` | integer | `0` for success |
| `statusMessage` | string | Human-readable status |
| `interfaceDetails.status` | string | Interface administrative state: `UP`, `DOWN` |
| `linkStatus.status` | string | Physical link state: `LINKED`, `NOT_LINKED` |
| `linkStatus.linkSpeed` | string | Negotiated link speed (e.g., `100Mbps`, `1Gbps`) |
| `ipv4Configuration.ipAddress` | string | Assigned IPv4 address |
| `ipv4Configuration.subnetMask` | string | Subnet mask |
| `ipv4Configuration.enableDhcp` | boolean | `true` if address is obtained via DHCP |
| `ipv4Configuration.gateway` | string | Default gateway address |
| `ipv4Configuration.dns1` | string | Primary DNS server |
| `ipv4Configuration.dns2` | string | Secondary DNS server |

## Error Responses

| Code | Name | Description |
|---|---|---|
| `1` | GeneralError | Unexpected internal error |
| `4` | InvalidRequestId | `requestId` is missing or empty |

## Notes

- When the Ethernet interface is not connected, `linkStatus.status` returns `NOT_LINKED` and `ipv4Configuration` fields may be empty or contain the last-known values.
- Ethernet connectivity is available when the device is docked in a cradle with an Ethernet connection.
- DHCP is the default configuration. Static IP assignment is configured via `set_config`.
