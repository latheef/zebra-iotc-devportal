# get_installed_certificate

Retrieves a list of all certificates currently installed on the device, including their type, expiry, issuer, and subject.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

```json
{
  "command": "get_installed_certificate",
  "requestId": "9"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_installed_certificate"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Response Payload

```json
{
  "command": "get_installed_certificate",
  "requestId": "9",
  "statusCode": 0,
  "statusMessage": "Success",
  "certificates": [
    {
      "name": "CorpRootCA",
      "type": "CA",
      "expiry": "2027-12-31T23:59:59Z",
      "issuer": "CN=Corporate Root CA, O=Acme Corp",
      "subject": "CN=Corporate Root CA, O=Acme Corp"
    },
    {
      "name": "MQTTBrokerCert",
      "type": "MQTT",
      "expiry": "2026-06-15T12:00:00Z",
      "issuer": "CN=Acme Intermediate CA, O=Acme Corp",
      "subject": "CN=mqtt.acme.com"
    },
    {
      "name": "DeviceClientCert",
      "type": "CLIENT",
      "expiry": "2026-11-30T23:59:59Z",
      "issuer": "CN=Acme Intermediate CA, O=Acme Corp",
      "subject": "CN=24042AR50033, O=Acme Corp"
    }
  ]
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `statusCode` | integer | `0` for success |
| `statusMessage` | string | Human-readable status |
| `certificates` | array | List of installed certificates |
| `certificates[].name` | string | Certificate alias name (used to reference the certificate in other commands) |
| `certificates[].type` | string | Certificate purpose: `CA`, `CLIENT`, `MQTT`, `WIFI` |
| `certificates[].expiry` | string | Expiration timestamp in ISO 8601 format |
| `certificates[].issuer` | string | Certificate issuer distinguished name |
| `certificates[].subject` | string | Certificate subject distinguished name |

## Error Responses

| Code | Name | Description |
|---|---|---|
| `1` | GeneralError | Unexpected internal error |
| `4` | InvalidRequestId | `requestId` is missing or empty |

## Notes

- The `certificates` array is empty if no certificates are installed.
- Certificate names are case-sensitive and must be unique on the device.
- The `type` field indicates where the certificate is used: `CA` for trust chain validation, `CLIENT` for mutual TLS, `MQTT` for MQTT broker TLS, and `WIFI` for 802.1X enterprise Wi-Fi.
- Private keys associated with `CLIENT` certificates are not exposed in the response.
