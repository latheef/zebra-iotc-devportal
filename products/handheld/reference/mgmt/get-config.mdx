# get_config

Retrieves the complete configuration document from the device, including network settings, MQTT endpoints, event configuration, and RFID operating parameters.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

```json
{
  "command": "get_config",
  "requestId": "16"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_config"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Response Payload

```json
{
  "command": "get_config",
  "requestId": "16",
  "statusCode": 0,
  "statusMessage": "Success",
  "config": {
    "deviceInfo": {
      "model": "RFD90",
      "serialNumber": "24042AR50033",
      "firmwareVersion": "2.1.0.0"
    },
    "network": {
      "ethernet": {
        "ipv4Configuration": {
          "enableDhcp": true
        }
      },
      "wifi": {
        "profiles": []
      }
    },
    "mqtt": {
      "activeEndpoint": "Production-Broker",
      "endpoints": []
    },
    "events": {
      "heartbeat": {
        "enabled": true,
        "intervalSeconds": 300
      },
      "alerts": {
        "enabled": true,
        "batteryLow": {
          "enabled": true,
          "thresholdPercent": 20
        }
      }
    },
    "rfid": {
      "operatingMode": {
        "mode": "INVENTORY",
        "transmitPowerIndex": 270,
        "session": "S0",
        "tagPopulation": 100
      },
      "postFilter": {
        "rules": []
      }
    }
  }
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `statusCode` | integer | `0` for success |
| `statusMessage` | string | Human-readable status |
| `config` | object | Complete device configuration document |
| `config.deviceInfo` | object | Read-only device identification |
| `config.network` | object | Ethernet and Wi-Fi configuration |
| `config.mqtt` | object | MQTT endpoint configurations |
| `config.events` | object | Event and alert configuration |
| `config.rfid` | object | RFID operating mode and filter configuration |

## Error Responses

| Code | Name | Description |
|---|---|---|
| `1` | GeneralError | Unexpected internal error |
| `4` | InvalidRequestId | `requestId` is missing or empty |

## Notes

- The response payload can be large depending on the number of configured Wi-Fi profiles and MQTT endpoints. Plan for payloads up to 16 KB.
- Sensitive data (passwords, passphrases, private keys) is not included in the configuration dump.
- The `config` object returned by `get_config` can be modified and sent back using `set_config` to apply changes. Only include the fields you want to modify.
- The `config.deviceInfo` section is read-only and is ignored if included in a `set_config` request.
