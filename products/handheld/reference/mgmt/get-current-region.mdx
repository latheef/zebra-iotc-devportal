# get_current_region

Retrieves the regulatory region, maximum transmit power, and RF channel configuration for the device.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

```json
{
  "command": "get_current_region",
  "requestId": "3"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_current_region"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Response Payload

```json
{
  "command": "get_current_region",
  "requestId": "3",
  "statusCode": 0,
  "statusMessage": "Success",
  "country": "United States",
  "regulatoryStandard": "FCC Part 15.247",
  "maxTxPowerSupported": 30.0,
  "lbtEnabled": false,
  "frequencyHopping": true,
  "channelData": [
    {
      "channelIndex": 1,
      "frequency": 902.75,
      "enabled": true
    },
    {
      "channelIndex": 2,
      "frequency": 903.25,
      "enabled": true
    }
  ]
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `statusCode` | integer | `0` for success |
| `statusMessage` | string | Human-readable status |
| `country` | string | Country name for the configured region |
| `regulatoryStandard` | string | Regulatory standard (e.g., `FCC Part 15.247`, `ETSI EN 302 208`) |
| `maxTxPowerSupported` | number | Maximum transmit power in dBm supported by the region |
| `lbtEnabled` | boolean | Whether Listen Before Talk is enabled (required in ETSI regions) |
| `frequencyHopping` | boolean | Whether frequency hopping spread spectrum (FHSS) is enabled |
| `channelData` | array | List of available RF channels |
| `channelData[].channelIndex` | integer | Channel number |
| `channelData[].frequency` | number | Center frequency in MHz |
| `channelData[].enabled` | boolean | Whether the channel is active |

## Error Responses

| Code | Name | Description |
|---|---|---|
| `1` | GeneralError | Unexpected internal error |
| `4` | InvalidRequestId | `requestId` is missing or empty |

## Notes

- The region is set at the factory based on the device SKU and cannot be changed via MQTT commands.
- `maxTxPowerSupported` represents the regulatory maximum. The actual transmit power used during RFID operations is configured via `set_operating_mode`.
- In ETSI regions, `lbtEnabled` is always `true` and `frequencyHopping` may be `false` depending on the specific regulatory standard.
- The `channelData` array length varies by region: FCC regions typically have 50 channels, ETSI regions have 4–10.
