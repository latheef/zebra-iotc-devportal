# get_version

Retrieves the firmware version, hardware model, serial number, and detailed component versions from the device.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

```json
{
  "command": "get_version",
  "requestId": "1"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_version"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Response Payload

```json
{
  "command": "get_version",
  "requestId": "1",
  "statusCode": 0,
  "statusMessage": "Success",
  "model": "RFD90",
  "serialNumber": "24042AR50033",
  "sku": "RFD9030-G21B700-US",
  "firmwareVersion": "2.1.0.0",
  "detailedVersions": {
    "iotcVersion": "1.3.2.0",
    "radioVersion": "3.3.2.1",
    "scannerVersion": "PAABLS00-006-R02"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `statusCode` | integer | `0` for success |
| `statusMessage` | string | Human-readable status |
| `model` | string | Hardware model (e.g., `RFD40`, `RFD90`) |
| `serialNumber` | string | Device serial number |
| `sku` | string | Full SKU part number |
| `firmwareVersion` | string | Overall firmware version |
| `detailedVersions` | object | Component-level version breakdown |
| `detailedVersions.iotcVersion` | string | IoT Connector agent version |
| `detailedVersions.radioVersion` | string | RFID radio firmware version |
| `detailedVersions.scannerVersion` | string | Barcode scanner firmware version |

## Error Responses

| Code | Name | Description |
|---|---|---|
| `1` | GeneralError | Unexpected internal error retrieving version data |
| `4` | InvalidRequestId | `requestId` is missing or empty |

## Notes

- This is a read-only, stateless query with no side effects.
- Use `get_version` after a firmware update (`set_os` + `reboot`) to confirm the new version is active.
- The `detailedVersions` object may include additional fields in future firmware releases.
