# config_events

Configures which asynchronous events the device publishes, heartbeat intervals, and alert thresholds for battery, temperature, and other conditions.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

### Enable All Events

```json
{
  "command": "config_events",
  "requestId": "abc123",
  "eventConfiguration": {
    "antenna": true,
    "terminalConnection": true,
    "firmwareUpdate": true,
    "gpi": true,
    "network": true,
    "exceptions": true,
    "ntp": true,
    "userApp": true,
    "heartbeat": true,
    "power": true,
    "battery": true,
    "temperature": true,
    "fileDownload": true,
    "cpuUsage": true,
    "flashUsage": true,
    "ramUsage": true,
    "heartbeatConfiguration": {
      "interval": 100,
      "inventoryStatus": true,
      "batteryStatus": true,
      "userApps": true
    },
    "cpuThreshold": 80,
    "ramThreshold": 80,
    "flashThreshold": 80,
    "temperatureThreshold": 55
  }
}
```

### Selective Event Configuration

```json
{
  "command": "config_events",
  "requestId": "abc123",
  "eventConfiguration": {
    "terminalConnection": true,
    "firmwareUpdate": true,
    "network": true,
    "heartbeat": true,
    "power": true,
    "battery": true,
    "temperature": true,
    "fileDownload": true,
    "heartbeatConfiguration": {
      "interval": 300,
      "inventoryStatus": true,
      "batteryStatus": true
    }
  }
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"config_events"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |
| `eventConfiguration` | object | Yes | Event enable flags and threshold configuration |
| `eventConfiguration.antenna` | boolean | No | Enable antenna connect/disconnect events |
| `eventConfiguration.terminalConnection` | boolean | No | Enable terminal connection/disconnection events |
| `eventConfiguration.firmwareUpdate` | boolean | No | Enable firmware update progress events |
| `eventConfiguration.gpi` | boolean | No | Enable General Purpose Input (GPI) state change events |
| `eventConfiguration.network` | boolean | No | Enable network interface events |
| `eventConfiguration.exceptions` | boolean | No | Enable exception and error events |
| `eventConfiguration.ntp` | boolean | No | Enable NTP synchronization events |
| `eventConfiguration.userApp` | boolean | No | Enable user application lifecycle events |
| `eventConfiguration.heartbeat` | boolean | No | Enable periodic heartbeat messages |
| `eventConfiguration.power` | boolean | No | Enable power source change events |
| `eventConfiguration.battery` | boolean | No | Enable battery status alert events |
| `eventConfiguration.temperature` | boolean | No | Enable temperature monitoring events |
| `eventConfiguration.fileDownload` | boolean | No | Enable file download progress events |
| `eventConfiguration.cpuUsage` | boolean | No | Enable CPU usage alert events |
| `eventConfiguration.flashUsage` | boolean | No | Enable flash storage usage alert events |
| `eventConfiguration.ramUsage` | boolean | No | Enable RAM usage alert events |
| `eventConfiguration.heartbeatConfiguration.interval` | integer | No | Heartbeat interval in seconds |
| `eventConfiguration.heartbeatConfiguration.inventoryStatus` | boolean | No | Include inventory status in heartbeat payload |
| `eventConfiguration.heartbeatConfiguration.batteryStatus` | boolean | No | Include battery status in heartbeat payload |
| `eventConfiguration.heartbeatConfiguration.userApps` | boolean | No | Include user application status in heartbeat payload |
| `eventConfiguration.cpuThreshold` | integer | No | CPU usage percentage threshold for alerts |
| `eventConfiguration.ramThreshold` | integer | No | RAM usage percentage threshold for alerts |
| `eventConfiguration.flashThreshold` | integer | No | Flash usage percentage threshold for alerts |
| `eventConfiguration.temperatureThreshold` | integer | No | Temperature threshold in Celsius for alerts |

## Response Payload

```json

  "command": "config_events",
  "requestId": "abc123",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "Success"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `apiVersion` | string | API version: `"V1.0"` or `"V1.1"` |
| `response` | object | Standard response envelope |
| `response.code` | integer | `0` for success |
| `response.description` | string | Human-readable result description |

## Error Responses

| Code | Description |
|---|---|
| `0` | Success |
| `1` | Failure |

## Notes

- Only the fields included in the request are updated. Omitted fields retain their current values.
- Alert events are published to the MGMT event topic: `{tenantId}/MGMT/clients/event/{deviceSerial}`.
- Event configuration persists across reboots.
- If `heartbeat` is `true` but `heartbeatConfiguration.interval` is omitted, the current interval setting is unchanged.
- Threshold fields (`cpuThreshold`, `ramThreshold`, `flashThreshold`, `temperatureThreshold`) only apply when their corresponding event flag is enabled.
