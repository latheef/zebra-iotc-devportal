# get_status

Queries the current operational status of the device including radio activity, power source, battery health, NTP synchronization, and terminal connection state.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Request Payload

```json
{
  "command": "get_status",
  "requestId": "abc123"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_status"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Error Responses

| Code | Description |
|---|---|
| `0` | Success |
| `3` | Not able to retrieve information |

## Notes

- `radioActivity` reflects real-time RFID radio state. Check for `INACTIVE` before starting a new inventory operation.
- `chargePercentage` is approximate. Actual capacity varies with battery age and temperature.
- `ntp.reach` â€” a non-zero value means NTP is reachable and syncing. A value of `0` means the NTP server is not reachable.
- Poll `get_status` to verify device readiness before starting operations. For continuous monitoring, configure `heartBeatEVT` instead.
