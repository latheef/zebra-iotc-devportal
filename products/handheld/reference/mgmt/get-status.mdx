# get_status

Queries the current operational status of the device including radio activity, power source, battery health, and NTP synchronization.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

```json
{
  "command": "get_status",
  "requestId": "2"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_status"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Response Payload

```json
{
  "command": "get_status",
  "requestId": "2",
  "statusCode": 0,
  "statusMessage": "Success",
  "radioActivity": "IDLE",
  "radioConnection": "CONNECTED",
  "powerSource": "BATTERY",
  "chargePercentage": 85,
  "stateOfHealth": "GOOD",
  "ntp": {
    "reach": 377,
    "status": "SYNCHRONIZED"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `statusCode` | integer | `0` for success |
| `statusMessage` | string | Human-readable status |
| `radioActivity` | string | RFID radio state: `IDLE`, `READING`, `WRITING`, `LOCATING` |
| `radioConnection` | string | Connection to host device: `CONNECTED`, `DISCONNECTED` |
| `powerSource` | string | Current power source: `BATTERY`, `CRADLE`, `USB` |
| `chargePercentage` | integer | Battery charge level (0–100) |
| `stateOfHealth` | string | Battery health: `GOOD`, `DEGRADED`, `REPLACE` |
| `ntp` | object | NTP synchronization details |
| `ntp.reach` | integer | NTP reach register (octal bitmask of last 8 polls, `377` = all successful) |
| `ntp.status` | string | `SYNCHRONIZED`, `NOT_SYNCHRONIZED`, `NOT_CONFIGURED` |

## Error Responses

| Code | Name | Description |
|---|---|---|
| `1` | GeneralError | Unexpected internal error |
| `4` | InvalidRequestId | `requestId` is missing or empty |

## Notes

- `radioActivity` reflects real-time RFID radio state. If a read operation is in progress, the value is `READING`.
- `chargePercentage` is approximate. Actual capacity varies with battery age and temperature.
- The `ntp.reach` value `377` (octal) indicates all eight most recent NTP polls succeeded. A value of `0` means no polls have succeeded.
- Poll `get_status` periodically to monitor device health in fleet management scenarios. For automated monitoring, configure `heartBeatEVT` events instead.
