# alert_short

Compact alert format optimized for Mobile Device Management (MDM) systems. Provides essential alert information in a minimal payload for bandwidth-constrained environments.

## Topic

| | Topic |
|---|---|
| **Event** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |

> **Note:** `alert_short` events are published on the same MGMT event topic as full `alerts`. MDM systems can filter by the `event` field to process only compact alerts.

## Direction

Reader → Application

## Payload

```json
{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747218125,
  "type": "BAT_LOW",
  "sev": "W",
  "val": 18,
  "thr": 20
}
```

```json
{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747221030,
  "type": "BAT_CRIT",
  "sev": "C",
  "val": 4,
  "thr": 5
}
```

```json
{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747214700,
  "type": "TEMP_HI",
  "sev": "W",
  "val": 52,
  "thr": 50
}
```

```json
{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747210800,
  "type": "PWR_CHG",
  "sev": "I",
  "val": 1,
  "thr": 0
}
```

| Field | Type | Description |
|---|---|---|
| `event` | string | Always `"alert_short"` |
| `sn` | string | Device serial number (abbreviated field name) |
| `ts` | integer | Unix timestamp (seconds since epoch) |
| `type` | string | Short alert type code (see table below) |
| `sev` | string | Severity code: `I` (info), `W` (warning), `C` (critical) |
| `val` | number | Current value that triggered the alert |
| `thr` | number | Configured threshold |

### Alert Type Codes

| Code | Full Alert Type | Description |
|---|---|---|
| `BAT_LOW` | `BATTERY_LOW` | Battery below low threshold |
| `BAT_CRIT` | `BATTERY_CRITICAL` | Battery below critical threshold |
| `TEMP_HI` | `TEMPERATURE_HIGH` | Temperature above threshold |
| `NET_DC` | `NETWORK_DISCONNECT` | Network interface disconnected |
| `PWR_CHG` | `POWER_SOURCE_CHANGE` | Power source changed |

### Severity Codes

| Code | Meaning |
|---|---|
| `I` | Info |
| `W` | Warning |
| `C` | Critical |

## Notes

- `alert_short` uses abbreviated field names and Unix timestamps to minimize payload size, typically under 100 bytes per alert.
- The `val` field semantics depend on the alert type: battery percentage for `BAT_LOW`/`BAT_CRIT`, temperature in Celsius for `TEMP_HI`, and a numeric indicator for `PWR_CHG` and `NET_DC`.
- For `PWR_CHG`, `val` encodes the new power source: `0` = CRADLE, `1` = BATTERY, `2` = USB.
- `alert_short` events are generated alongside full `alerts` events. Both are published when an alert condition is triggered.
- MDM systems that do not need the full `alerts` payload can subscribe to the same topic and filter for `"event": "alert_short"`.
