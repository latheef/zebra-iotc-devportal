# alert_short

Compact alert notification for lightweight event logging and real-time monitoring. Provides a short identifier, type, priority, and human-readable description for each alert condition.

## Topic

| | Topic |
|---|---|
| **Event** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |

## Payload

### Firmware Download Success

```json
{
  "id": "FIRMWARE_DOWNLOAD_SUCCESS",
  "timestamp": "22:54:25",
  "type": "ALERT",
  "priority": "LOW",
  "messageID": "ZEBRA_RFID_HH_ALERTS",
  "description": "firmware download completed successfully"
}
```

### Battery Critical

```json
{
  "id": "BATTERY_CRITICAL_SET",
  "timestamp": "14:30:15",
  "type": "ALERT",
  "priority": "CRITICAL",
  "messageID": "ZEBRA_RFID_HH_ALERTS",
  "description": "battery level is critically low"
}
```

### Wi-Fi Configuration Failure

```json
{
  "id": "WIFI_CONFIG_FAIL",
  "timestamp": "09:15:42",
  "type": "NOTIFICATION",
  "priority": "HIGH",
  "messageID": "ZEBRA_RFID_HH_ALERTS",
  "description": "WiFi configuration failed"
}
```

### MQTT CA Certificate Install Success

```json
{
  "id": "MQTT_ROOT_CERT_INSTALL_SUCCESS",
  "timestamp": "2026-04-17T05:09:13.808Z",
  "type": "NOTIFICATION",
  "priority": "HIGH",
  "messageID": "ZEBRA_RFID_HH_ALERTS",
  "description": "MQTT root certificate install success"
}
```

| Field | Type | Description |
|---|---|---|
| `id` | string | Alert identifier (see full enum table below) |
| `timestamp` | string | Time when the alert was generated |
| `type` | string | Category: `NOTIFICATION` or `ALERT` |
| `priority` | string | Urgency level: `CRITICAL`, `HIGH`, or `LOW` |
| `messageID` | string | Always `"ZEBRA_RFID_HH_ALERTS"` |
| `description` | string | Human-readable description of the alert condition |

## Notes

- `CRITICAL` priority alerts (e.g., `BATTERY_CRITICAL_SET`) require immediate attention.
- `LOW` priority alerts (e.g., `FIRMWARE_DOWNLOAD_SUCCESS`) are informational.
- `alert_short` events are published on the same MGMT event topic as full `alerts` events. Filter by event content to distinguish them.
- The `messageID` field is always `"ZEBRA_RFID_HH_ALERTS"` and can be used to identify this event type.
