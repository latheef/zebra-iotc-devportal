# alerts

Detailed, structured alert events published when device conditions transition state or cross monitored thresholds. Covers battery, power, firmware, temperature, and network conditions.

## Topic

| | Topic |
|---|---|
| **Event** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |

## Direction

Reader → Application

## Payload

### Battery Alert

```json
{
  "type": "ALERT",
  "timestamp": "2026-04-29T12:33:34.279Z",
  "state": "ONESHOT",
  "id": "BATTERY",
  "priority": "LOW",
  "alertDetails": {
    "batteryAlert": {
      "status": "CHARGING",
      "stateOfHealth": "GOOD",
      "chargePercentage": 100
    }
  }
}
```

### Firmware Update in Progress

```json
{
  "type": "ALERT",
  "timestamp": "2026-04-29T12:33:57.757Z",
  "state": "SET",
  "id": "FIRMWARE_UPDATE",
  "priority": "CRITICAL",
  "alertDetails": {
    "fwUpdateStatus": {
      "updateStatus": "updating",
      "overallProgress": 20,
      "stage": "updating scanner fw"
    }
  }
}
```

| Field | Type | Description |
|---|---|---|
| `type` | string | Message category: `EVENT`, `NOTIFICATION`, or `ALERT` |
| `timestamp` | string | Alert timestamp |
| `state` | string | Alert state (see table below) |
| `id` | string | Alert type identifier (see table below) |
| `priority` | string | Priority: `CRITICAL`, `HIGH`, `MEDIUM`, or `LOW` |
| `alertDetails` | object | Domain-specific details (structure varies by `id`) |

### Alert States

| State | Description |
|---|---|
| `SET` | Alert condition is currently active |
| `CLEAR` | Alert condition has been resolved |
| `ONESHOT` | One-time event — fires once, no corresponding CLEAR |

### Alert Identifiers

| `id` | `state` Values | Trigger Condition |
|---|---|---|
| `BATTERY` | `ONESHOT` | Battery charge or health state changes |
| `POWER` | `ONESHOT` | Power source changes (e.g., USB to battery) |
| `NETWORK_EVENT` | `ONESHOT` | Wi-Fi or Ethernet interface connects, disconnects, or changes IP |
| `FIRMWARE_UPDATE` | `SET` / `CLEAR` | Firmware update starts (`SET`) or completes/fails (`CLEAR`) |
| `TEMPERATURE` | `SET` / `CLEAR` | Temperature crosses threshold (`SET`) or returns to normal (`CLEAR`) |

### alertDetails Fields by Alert Type

**`id: BATTERY`** — `alertDetails.batteryAlert`:

| Field | Type | Description |
|---|---|---|
| `status` | string | Battery state: `LOW`, `CRITICAL`, `CHARGING`, `FULL`, `HIGH` |
| `stateOfHealth` | string | Battery health: `LOW`, `CRITICAL`, `CHARGING`, `FULL`, `HIGH` |
| `chargePercentage` | integer | Battery charge percentage (0–100) |

**`id: POWER`** — `alertDetails.powerEvent`:

| Field | Type | Description |
|---|---|---|
| `powerSource` | string | New power source (e.g., `BATTERY`, `USB`) |
| `powerMode` | string | Power mode (e.g., `ACTIVE`) |

**`id: NETWORK_EVENT`** — `alertDetails.networkInfo.networkInterface`:

| Field | Type | Description |
|---|---|---|
| `wifiStatus[]` | array | Wi-Fi interface status objects (`interface`, `status`, `ssid`, `ipV4Address`) |
| `ethStatus[]` | array | Ethernet interface status objects (`interface`, `status`, `linkStatus`, `ipV4Address`) |

**`id: FIRMWARE_UPDATE`** — `alertDetails.fwUpdateStatus`:

| Field | Type | Description |
|---|---|---|
| `updateStatus` | string | `started`, `updating`, `successfull`, `failed`, `skipped` |
| `overallProgress` | number | Percentage of update completed |
| `stage` | string | Description of the current update step |

**`id: TEMPERATURE`** — `alertDetails.temperatueInfo`:

| Field | Type | Description |
|---|---|---|
| `nge` | number | Next Generation Event value |
| `pa` | number | Pressure altitude value |
| `ambient` | number | Ambient temperature |

## Notes

- Alerts are enabled and configured via `config_events`. Each alert type can be independently enabled or disabled.
- `ONESHOT` events fire once when the condition occurs and do not require a corresponding `CLEAR`.
- `SET` / `CLEAR` pairs indicate persistent conditions — `SET` when active, `CLEAR` when resolved.
- Alert events are published on the MGMT event topic alongside `heartBeatEVT` and `mqttConnEVT`.
