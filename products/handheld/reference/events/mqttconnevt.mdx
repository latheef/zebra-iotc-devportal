# mqttConnEVT

Endpoint connectivity state change events published when the device connects to or disconnects from the MQTT broker. Includes device model, serial number, and protocol version context.

## Topic

| | Topic |
|---|---|
| **Event** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |

## Direction

Reader → Application

## Payload

### Connection Event

```json
{
  "connectionState": "CONNECTED",
  "timestamp": "12:17:56",
  "deviceModel": "RFD40",
  "deviceSerialNo": "RFD40-24190525100354",
  "apiVersion": "1.0",
  "mqttVersion": "3.1.1"
}
```

### Disconnection Event

```json
{
  "connectionState": "DISCONNECTED",
  "deviceModel": "RFD40",
  "deviceSerialNo": "RFD40-24190525100354",
  "apiVersion": 1.2,
  "mqttVersion": 3.1
}
```

| Field | Type | Description |
|---|---|---|
| `connectionState` | string | Connection status: `CONNECTED` or `DISCONNECTED` |
| `timestamp` | string | Time of the connection event (may be absent on `DISCONNECTED`) |
| `deviceModel` | string | Hardware model: `RFD40` or `RFD90` |
| `deviceSerialNo` | string | Unique serial number of the device |
| `apiVersion` | number | API version in use during the connection event |
| `mqttVersion` | number | MQTT protocol version (e.g., `3.1.1`) |

## Notes

- A `CONNECTED` event is published immediately after each successful connection or reconnection. `timestamp` is always present on `CONNECTED` events.
- A `DISCONNECTED` event is published when the connection drops. `timestamp` may be absent if the device cannot reach the broker at the time of disconnect.
- After a `DISCONNECTED` event, the device automatically attempts to reconnect using exponential backoff (initial 1 second, maximum 60 seconds).
- Disconnection events may be delivered via MQTT Last Will and Testament (LWT), depending on broker configuration.
- `mqttConnEVT` events are enabled or disabled via `config_events`. They are enabled by default.
- Monitor this event for fleet-wide connectivity tracking and to correlate device online/offline state with data gaps.
