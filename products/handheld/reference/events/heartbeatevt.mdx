# heartBeatEVT

Periodic liveness and health event published by the device at a configurable interval. Reports device uptime, inventory status, and battery health.

## Topic

| | Topic |
|---|---|
| **Event** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |

## Direction

Reader → Application

## Payload

### Heartbeat with Inventory and Battery Status

```json
{
  "eventName": "heartbeat",
  "timestamp": "2019-08-24T14:15:22Z",
  "eventNumber": 120,
  "upTime": "5 days 4hr 3min",
  "data": {
    "inventoryStatus": {
      "rfidStatus": "INPROGRESS",
      "tagCount": 45,
      "scanCount": 128
    },
    "batteryAlert": {
      "status": "HIGH",
      "stateOfHealth": "FULL",
      "chargePercentage": 85
    }
  }
}
```

### Heartbeat with Battery Alert Only

```json
{
  "eventName": "heartbeat",
  "timestamp": "2026-05-14T10:35:00.000Z",
  "eventNumber": 42,
  "upTime": "1 days 2hr 15min",
  "data": {
    "batteryAlert": {
      "status": "LOW",
      "stateOfHealth": "FULL",
      "chargePercentage": 18
    }
  }
}
```

| Field | Type | Description |
|---|---|---|
| `eventName` | string | Always `"heartbeat"` |
| `timestamp` | string | Event timestamp in ISO 8601 format |
| `eventNumber` | integer | Sequence number for the event |
| `upTime` | string | Device uptime since last boot (e.g., `"5 days 4hr 3min"`) |
| `data` | object | Heartbeat payload; contents depend on `heartbeatConfiguration` |
| `data.inventoryStatus` | object | Present when `inventoryStatus` is enabled in `heartbeatConfiguration` |
| `data.inventoryStatus.rfidStatus` | string | RFID inventory state: `INPROGRESS` or `STOPPED` |
| `data.inventoryStatus.tagCount` | integer | Total tags scanned in the current operation |
| `data.inventoryStatus.scanCount` | integer | Total scan attempts in the current operation |
| `data.batteryAlert` | object | Present when `batteryStatus` is enabled in `heartbeatConfiguration` |
| `data.batteryAlert.status` | string | Battery state: `LOW`, `CRITICAL`, `CHARGING`, `FULL`, or `HIGH` |
| `data.batteryAlert.stateOfHealth` | string | Battery health: `LOW`, `CRITICAL`, `CHARGING`, `FULL`, or `HIGH` |
| `data.batteryAlert.chargePercentage` | integer | Battery charge level (0–100) |

## Notes

- Heartbeat events are enabled and configured via `config_events` under `eventConfiguration.heartbeat` and `eventConfiguration.heartbeatConfiguration`.
- The `data.inventoryStatus` block is included only when `heartbeatConfiguration.inventoryStatus` is `true`.
- The `data.batteryAlert` block is included only when `heartbeatConfiguration.batteryStatus` is `true`.
- If the device loses MQTT connectivity, heartbeat events are not queued. The absence of heartbeats is itself an indicator of connectivity loss.
- Fleet management systems should monitor heartbeat events and raise alerts when a device misses multiple consecutive heartbeats.
