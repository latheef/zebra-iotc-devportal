# control_operation

Starts or stops an RFID or barcode scanner operation on the device using the currently configured operating mode.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/CTRL/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/CTRL/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

### Start RFID Inventory

```json
{
  "command": "control_operation",
  "requestId": "abcd1432",
  "ctrlOprPayload": {
    "controlType": "RFID",
    "operation": "START"
  }
}
```

### Stop RFID Inventory

```json
{
  "command": "control_operation",
  "requestId": "abcd1432",
  "ctrlOprPayload": {
    "controlType": "RFID",
    "operation": "STOP"
  }
}
```

### Start Barcode Scanner

```json
{
  "command": "control_operation",
  "requestId": "abcd1433",
  "ctrlOprPayload": {
    "controlType": "SCANNER",
    "operation": "START"
  }
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"control_operation"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |
| `ctrlOprPayload` | object | Yes | Specifies the control type and operation to perform |
| `ctrlOprPayload.controlType` | string | Yes | Subsystem to control: `RFID` or `SCANNER` |
| `ctrlOprPayload.operation` | string | Yes | Action to perform: `START` or `STOP` |

## Response Payload

```json
{
  "command": "control_operation",
  "requestId": "abcd1432",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "Success"
  }
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `apiVersion` | string | API version: `"V1.0"` or `"V1.1"` |
| `response` | object | Standard response envelope |
| `response.code` | integer | Response code (see table below) |
| `response.description` | string | Human-readable result description |

## Error Responses

| Code | Description |
|---|---|
| `0` | Success |
| `11` | Inventory in progress — cannot start while operation is already running |
| `12` | No radio operation in progress — STOP sent when already idle |
| `23` | Invalid enum value — `controlType` or `operation` field has an unrecognized value |

## Notes

- After `START`, tag read data is published to the DATA topics (`data1event`/`data2event`) as `dataEVT` events.
- Sending `STOP` when no operation is running returns error code `12`. This is not a failure — the device is already idle.
- Sending `START` when an operation is already running returns error code `11`. Stop the current operation first.
- The operating mode must be configured via `set_operating_mode` before starting an operation. If no mode has been set, the device uses its default configuration.
- RFID operations consume significant battery. Monitor `get_status` or configure battery alerts via `config_events`.
