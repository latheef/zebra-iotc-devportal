# get_post_filter

Retrieves the current post-read filter rules configured on the device. Post-filters determine which tag reads are reported in `dataEVT` events.

## Topic

| | Topic |
|---|---|
| **Request** | `{tenantId}/CTRL/clients/cmnd/{deviceSerial}` |
| **Response** | `{tenantId}/CTRL/clients/resp/{deviceSerial}` |

## Direction

Application → Reader → Application

## Request Payload

```json
{
  "command": "get_post_filter",
  "requestId": "31"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `command` | string | Yes | Must be `"get_post_filter"` |
| `requestId` | string | Yes | Unique identifier to correlate the response |

## Response Payload

```json
{
  "command": "get_post_filter",
  "requestId": "31",
  "statusCode": 0,
  "statusMessage": "Success",
  "postFilter": {
    "enabled": true,
    "rules": [
      {
        "ruleId": 1,
        "memoryBank": "EPC",
        "offset": 32,
        "length": 16,
        "pattern": "3034",
        "matchType": "INCLUDE",
        "enabled": true
      },
      {
        "ruleId": 2,
        "memoryBank": "EPC",
        "offset": 32,
        "length": 16,
        "pattern": "E200",
        "matchType": "EXCLUDE",
        "enabled": true
      }
    ]
  }
}
```

| Field | Type | Description |
|---|---|---|
| `command` | string | Echo of the command name |
| `requestId` | string | Echo of the request identifier |
| `statusCode` | integer | `0` for success |
| `statusMessage` | string | Human-readable status |
| `postFilter.enabled` | boolean | Whether post-filtering is globally active |
| `postFilter.rules` | array | List of configured filter rules |
| `postFilter.rules[].ruleId` | integer | Unique rule identifier |
| `postFilter.rules[].memoryBank` | string | Memory bank to match against: `EPC`, `TID`, `USER` |
| `postFilter.rules[].offset` | integer | Bit offset from the start of the memory bank |
| `postFilter.rules[].length` | integer | Number of bits to compare |
| `postFilter.rules[].pattern` | string | Hexadecimal pattern to match |
| `postFilter.rules[].matchType` | string | `INCLUDE` (report matching tags) or `EXCLUDE` (suppress matching tags) |
| `postFilter.rules[].enabled` | boolean | Whether this rule is active |

## Error Responses

| Code | Name | Description |
|---|---|---|
| `1` | GeneralError | Unexpected internal error |
| `4` | InvalidRequestId | `requestId` is missing or empty |

## Notes

- If no filter rules are configured, `postFilter.rules` is an empty array and all tag reads are reported.
- Filters are applied in order. An `INCLUDE` rule passes matching tags, an `EXCLUDE` rule suppresses them.
- Filter rules operate on the raw memory bank data. The `offset` and `length` are in bits, not bytes.
- Filters can be modified while an RFID operation is in progress, but the new rules take effect only after stopping and restarting the operation.
