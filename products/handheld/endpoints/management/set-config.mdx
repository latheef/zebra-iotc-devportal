# Set Config

Set multiple device configuration domains in a single command on a Zebra handheld RFID reader. The payload can include any combination of Wi-Fi, endpoint, event, and certificate configuration. Only the domains included in the request are modified; omitted domains are left unchanged.

## Endpoint Details

| Property | Value |
|----------|-------|
| **Command** | `set_config` |
| **Interface** | MGMT |
| **Direction** | PUB (command) → SUB (response) |
| **Command Topic** | `{tenantId}/MGMT/clients/cmnd/{deviceSerial}` |
| **Response Topic** | `{tenantId}/MGMT/clients/resp/{deviceSerial}` |
| **Content Type** | `application/json` |

## Request Payload (PUB)

```json
{
  "command": "set_config",
  "requestId": "req-001",
  "configPayload": {
    "wifiConfig": {
      "operation": "ADD",
      "ssid": "Warehouse-5G",
      "securityType": "WPA2_PERSONAL",
      "password": "securePass123"
    },
    "endpointConfig": {
      "mqttBroker": "mqtt.example.com",
      "mqttPort": 8883
    },
    "eventConfig": {
      "tagReadReporting": true,
      "batteryAlerts": true
    },
    "certificates": [
      {
        "name": "ca-root",
        "type": "CA",
        "data": "-----BEGIN CERTIFICATE-----\nMIID...base64...\n-----END CERTIFICATE-----"
      }
    ]
  }
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `command` | string | Yes | Must be `"set_config"`. |
| `requestId` | string | Yes | Unique identifier to correlate the response with this request. |
| `configPayload` | object | Yes | Container for configuration domains to set. |
| `configPayload.wifiConfig` | object | No | Wi-Fi profile configuration. See [Set Wi-Fi](set-wifi) for field details. |
| `configPayload.endpointConfig` | object | No | Cloud endpoint connection settings. |
| `configPayload.eventConfig` | object | No | Event reporting and filtering settings. |
| `configPayload.certificates` | array | No | Array of certificate objects to install. Each object includes `name`, `type`, and `data` fields. |

## Response Payload (SUB)

```json
{
  "command": "set_config",
  "requestId": "req-001",
  "responseCode": 0
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `command` | string | Echo of the command name. |
| `requestId` | string | Echo of the request identifier. |
| `responseCode` | integer | Result code indicating success or failure. |

## Response Codes

| Code | Description |
|------|-------------|
| `0` | Success. |
| `1` | General failure. |
| `2` | Invalid payload; the request body is malformed or contains invalid values. |
