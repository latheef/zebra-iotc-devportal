# Set Post Filter

Configure tag data post-filters on the reader. Post-filters are applied after tag reads and allow you to include or exclude tags based on EPC prefix, suffix, regular expression patterns, RSSI threshold, or explicit whitelist/blacklist of tag identifiers.

## Endpoint Details

| Property | Value |
|----------|-------|
| **Command** | `set_post_filter` |
| **Interface** | CTRL |
| **Direction** | PUB (command) → SUB (response) |
| **Command Topic** | `{tenantId}/CTRL/clients/cmnd/{deviceSerial}` |
| **Response Topic** | `{tenantId}/CTRL/clients/resp/{deviceSerial}` |
| **Content Type** | `application/json` |

## Request Payload (PUB)

Publish a JSON message with the post-filter configuration. The fields required depend on the `filterType` selected.

```json
{
  "command": "set_post_filter",
  "requestId": "unique-id",
  "postFilter": {
    "filterType": "PREFIX",
    "action": "INCLUDE",
    "pattern": "E200",
    "enable": true
  }
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `command` | string | Yes | Must be `"set_post_filter"`. |
| `requestId` | string | Yes | Unique identifier to correlate request and response. |
| `postFilter` | object | Yes | Post-filter configuration object. |
| `postFilter.filterType` | enum | Yes | Type of filter. One of: `PREFIX`, `SUFFIX`, `REGEX`, `RSSI`, `WHITELIST`, `BLACKLIST`. |
| `postFilter.action` | enum | Yes | Filter action. One of: `INCLUDE`, `EXCLUDE`. |
| `postFilter.pattern` | string | Conditional | Match pattern (max 64 characters). Required when `filterType` is `PREFIX`, `SUFFIX`, or `REGEX`. |
| `postFilter.rssiThreshold` | integer | Conditional | RSSI threshold in dBm (range: `-100` to `0`). Required when `filterType` is `RSSI`. |
| `postFilter.tagList` | array of strings | Conditional | List of tag EPC identifiers. Required when `filterType` is `WHITELIST` or `BLACKLIST`. |
| `postFilter.enable` | boolean | Yes | Enable or disable the post-filter. |

### Filter Type Usage

| `filterType` | Required Fields | Description |
|--------------|----------------|-------------|
| `PREFIX` | `pattern` | Match tags whose EPC starts with the given pattern. |
| `SUFFIX` | `pattern` | Match tags whose EPC ends with the given pattern. |
| `REGEX` | `pattern` | Match tags whose EPC matches the regular expression. |
| `RSSI` | `rssiThreshold` | Filter tags by minimum RSSI signal strength. |
| `WHITELIST` | `tagList` | Only include/exclude tags in the provided list. |
| `BLACKLIST` | `tagList` | Only include/exclude tags in the provided list. |

## Response Payload (SUB)

```json
{
  "command": "set_post_filter",
  "requestId": "unique-id",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "SUCCESS"
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `command` | string | Echo of the command name. |
| `requestId` | string | Echo of the request identifier. |
| `apiVersion` | string | API version (e.g., `"V1.1"`). |
| `response.code` | integer | Numeric response code. |
| `response.description` | string | Human-readable description of the result. |

## Response Codes

| Code | Description |
|------|-------------|
| `0` | SUCCESS |
| `1` | FAILURE |
| `2` | Invalid payload |
| `28` | Pattern exceeds maximum length (64 characters) |

### Request: Prefix Filter

```json
{
  "command": "set_post_filter",
  "requestId": "req-005",
  "postFilter": {
    "filterType": "PREFIX",
    "action": "INCLUDE",
    "pattern": "E200",
    "enable": true
  }
}
```

### Response: Success

```json
{
  "command": "set_post_filter",
  "requestId": "req-005",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "SUCCESS"
  }
}
```
