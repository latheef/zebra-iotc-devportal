# Get Post Filter

Retrieve the current post-filter configuration from the reader. The response includes the active filter type, action, pattern or tag list, and enabled status. If no post-filter has been configured, the `postFilter` object is omitted from the response.

## Endpoint Details

| Property | Value |
|----------|-------|
| **Command** | `get_post_filter` |
| **Interface** | CTRL |
| **Direction** | PUB (command) → SUB (response) |
| **Command Topic** | `{tenantId}/CTRL/clients/cmnd/{deviceSerial}` |
| **Response Topic** | `{tenantId}/CTRL/clients/resp/{deviceSerial}` |
| **Content Type** | `application/json` |

## Request Payload (PUB)

Send a simple command payload with no additional configuration fields.

```json
{
  "command": "get_post_filter",
  "requestId": "unique-id"
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `command` | string | Yes | Must be `"get_post_filter"`. |
| `requestId` | string | Yes | Unique identifier to correlate request and response. |

## Response Payload (SUB)

On success, the response includes the `postFilter` object with the current filter configuration. If no filter is configured, `postFilter` is omitted.

```json
{
  "command": "get_post_filter",
  "requestId": "unique-id",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "SUCCESS"
  },
  "postFilter": {
    "filterType": "PREFIX",
    "action": "INCLUDE",
    "pattern": "E200",
    "enable": true
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `command` | string | Echo of the command name. |
| `requestId` | string | Echo of the request identifier. |
| `apiVersion` | string | API version (e.g., `"V1.1"`). |
| `response.code` | integer | Numeric response code. |
| `response.description` | string | Human-readable description of the result. |
| `postFilter` | object | Current post-filter configuration. Omitted if no filter is configured. |
| `postFilter.filterType` | enum | Active filter type. One of: `PREFIX`, `SUFFIX`, `REGEX`, `RSSI`, `WHITELIST`, `BLACKLIST`. |
| `postFilter.action` | enum | Filter action. One of: `INCLUDE`, `EXCLUDE`. |
| `postFilter.pattern` | string | Match pattern. Present when `filterType` is `PREFIX`, `SUFFIX`, or `REGEX`. |
| `postFilter.rssiThreshold` | integer | RSSI threshold in dBm. Present when `filterType` is `RSSI`. |
| `postFilter.tagList` | array of strings | List of tag EPCs. Present when `filterType` is `WHITELIST` or `BLACKLIST`. |
| `postFilter.enable` | boolean | Whether the filter is currently enabled. |

## Response Codes

| Code | Description |
|------|-------------|
| `0` | SUCCESS |
| `1` | FAILURE |
| `3` | Unable to retrieve post-filter configuration |

## Example

### Request

```json
{
  "command": "get_post_filter",
  "requestId": "req-008"
}
```

### Response: Filter Configured

```json
{
  "command": "get_post_filter",
  "requestId": "req-008",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "SUCCESS"
  },
  "postFilter": {
    "filterType": "PREFIX",
    "action": "INCLUDE",
    "pattern": "E200",
    "enable": true
  }
}
```

### Response: No Filter Configured

```json
{
  "command": "get_post_filter",
  "requestId": "req-009",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "SUCCESS"
  }
}
```

### Response: RSSI Filter

```json
{
  "command": "get_post_filter",
  "requestId": "req-010",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "SUCCESS"
  },
  "postFilter": {
    "filterType": "RSSI",
    "action": "EXCLUDE",
    "rssiThreshold": -70,
    "enable": true
  }
}
```

### Response: Whitelist Filter

```json
{
  "command": "get_post_filter",
  "requestId": "req-011",
  "apiVersion": "V1.1",
  "response": {
    "code": 0,
    "description": "SUCCESS"
  },
  "postFilter": {
    "filterType": "WHITELIST",
    "action": "INCLUDE",
    "tagList": [
      "E20068060000004014B4E81E",
      "E20068060000004014B4E82F",
      "E20068060000004014B4E830"
    ],
    "enable": true
  }
}
```
