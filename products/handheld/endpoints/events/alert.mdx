# Alert Event

The `alerts` event is published when a monitored value crosses a configured threshold. Alerts cover temperature exceedances, battery degradation, and radio hardware errors, each with severity level and the specific values that triggered the alert.

## Endpoint Details

| Property | Value |
|----------|-------|
| **Event Name** | `alerts` |
| **Interface** | MGMT |
| **Direction** | SUB (subscribe to receive) |
| **Topic** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |
| **Content Type** | `application/json` |

## Event Payload (SUB)

Each alert message contains the alert type, severity, a human-readable message, and the current and threshold values that triggered the condition.

```json
{
  "event": "alerts",
  "deviceSerial": "string",
  "timestamp": "2026-04-14T12:00:00.000Z",
  "apiVersion": "string",
  "alert": {
    "alertType": "TEMPERATURE_HIGH",
    "severity": "WARNING",
    "message": "string",
    "currentValue": 0,
    "thresholdValue": 0,
    "unit": "string"
  }
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event` | string | Yes | Constant value `"alerts"`. |
| `deviceSerial` | string | Yes | Serial number of the reporting device. |
| `timestamp` | string (date-time) | Yes | ISO 8601 timestamp of the event. |
| `apiVersion` | string | Yes | API version identifier. |
| `alert` | object | Yes | Alert detail container. |
| `alert.alertType` | string | Yes | Type of alert. One of `TEMPERATURE_HIGH`, `BATTERY_LOW`, `BATTERY_CRITICAL`, `BATTERY_HEALTH_DEGRADED`, `RADIO_ERROR`. |
| `alert.severity` | string | Yes | Alert severity. One of `INFO`, `WARNING`, `CRITICAL`. |
| `alert.message` | string | Yes | Human-readable description of the alert condition. |
| `alert.currentValue` | number | Yes | The measured value that triggered the alert. |
| `alert.thresholdValue` | number | Yes | The configured threshold that was exceeded. |
| `alert.unit` | string | Yes | Unit of measurement (e.g., `"Â°C"`, `"%"`, `"count"`). |
