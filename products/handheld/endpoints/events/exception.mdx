# Exception Event

The `exceptionEVT` event is published when the device encounters a runtime error or hardware exception. This includes RFID radio communication failures, Bluetooth disconnections, network errors, MQTT publishing failures, firmware faults, memory errors, and general hardware errors.

## Endpoint Details

| Property | Value |
|----------|-------|
| **Event Name** | `exceptionEVT` |
| **Interface** | MGMT |
| **Direction** | SUB (subscribe to receive) |
| **Topic** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |
| **Content Type** | `application/json` |

## Event Payload (SUB)

Each exception message identifies the error type, affected component, error code, and a descriptive message. A stack trace may be included for firmware-level exceptions.

```json
{
  "event": "exceptionEVT",
  "deviceSerial": "string",
  "timestamp": "2026-04-14T12:00:00.000Z",
  "apiVersion": "string",
  "exception": {
    "exceptionType": "RADIO_COMMUNICATION_ERROR",
    "errorCode": 0,
    "message": "string",
    "component": "RFID_RADIO",
    "stackTrace": "string"
  }
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event` | string | Yes | Constant value `"exceptionEVT"`. |
| `deviceSerial` | string | Yes | Serial number of the reporting device. |
| `timestamp` | string (date-time) | Yes | ISO 8601 timestamp of the event. |
| `apiVersion` | string | Yes | API version identifier. |
| `exception` | object | Yes | Exception detail container. |
| `exception.exceptionType` | string | Yes | Type of exception. One of `RADIO_COMMUNICATION_ERROR`, `RADIO_INITIALIZATION_ERROR`, `BLUETOOTH_DISCONNECTION`, `NETWORK_ERROR`, `MQTT_PUBLISH_ERROR`, `FIRMWARE_ERROR`, `MEMORY_ERROR`, `HARDWARE_ERROR`. |
| `exception.errorCode` | integer | Yes | Numeric error code for programmatic handling. |
| `exception.message` | string | Yes | Human-readable error description. |
| `exception.component` | string | Yes | Affected hardware/software component. One of `RFID_RADIO`, `BLUETOOTH`, `NETWORK`, `SYSTEM`. |
| `exception.stackTrace` | string | No | Firmware stack trace, if available. |

## Example

```json
{
  "event": "exceptionEVT",
  "deviceSerial": "RFD40-212735201D0053",
  "timestamp": "2026-04-14T16:45:10.000Z",
  "apiVersion": "1.0",
  "exception": {
    "exceptionType": "RADIO_COMMUNICATION_ERROR",
    "errorCode": 3001,
    "message": "RFID radio module failed to respond within timeout period",
    "component": "RFID_RADIO",
    "stackTrace": "com.zebra.rfid.RadioManager.sendCommand(RadioManager.java:412)\ncom.zebra.rfid.RadioManager.inventory(RadioManager.java:298)"
  }
}
```
