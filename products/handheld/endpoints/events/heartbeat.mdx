# Heartbeat Event

The `heartBeatEVT` event is published periodically by the device to report its current health status. Each heartbeat includes uptime and temperature data, with optional battery status, inventory status, and network status sections that can be enabled or disabled via the `config_events` command.

## Endpoint Details

| Property | Value |
|----------|-------|
| **Event Name** | `heartBeatEVT` |
| **Interface** | MGMT |
| **Direction** | SUB (subscribe to receive) |
| **Topic** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |
| **Content Type** | `application/json` |

## Event Payload (SUB)

The heartbeat payload contains device identification fields and a `heartbeat` object with health metrics. Optional subsections are included only when enabled in the device's event configuration.

```json
{
  "event": "heartBeatEVT",
  "deviceSerial": "string",
  "timestamp": "2026-04-14T12:00:00.000Z",
  "apiVersion": "string",
  "heartbeat": {
    "uptimeSeconds": 0,
    "temperature": 0,
    "batteryStatus": { },
    "inventoryStatus": { },
    "networkStatus": { }
  }
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event` | string | Yes | Constant value `"heartBeatEVT"`. |
| `deviceSerial` | string | Yes | Serial number of the reporting device. |
| `timestamp` | string (date-time) | Yes | ISO 8601 timestamp of the event. |
| `apiVersion` | string | Yes | API version identifier. |
| `heartbeat` | object | Yes | Container for all health metrics. |
| `heartbeat.uptimeSeconds` | integer | Yes | Seconds elapsed since last device boot. |
| `heartbeat.temperature` | integer | Yes | Current device temperature in Celsius. |
| `heartbeat.batteryStatus` | object | No | Battery health data. Included when battery reporting is enabled. |
| `heartbeat.batteryStatus.stateOfHealth` | string | Yes | Battery condition. One of `GOOD`, `DEGRADED`, `REPLACE`. |
| `heartbeat.batteryStatus.chargePercentage` | integer | Yes | Current charge level (0–100). |
| `heartbeat.batteryStatus.chargeStatus` | integer | Yes | Charging state: `0` = not charging, `1` = charging, `2` = fully charged. |
| `heartbeat.batteryStatus.voltage` | integer | Yes | Battery voltage in millivolts. |
| `heartbeat.batteryStatus.temperature` | integer | Yes | Battery temperature in Celsius. |
| `heartbeat.inventoryStatus` | object | No | RFID radio activity data. Included when inventory reporting is enabled. |
| `heartbeat.inventoryStatus.radioActivity` | string | Yes | Radio state. One of `ACTIVE`, `INACTIVE`. |
| `heartbeat.inventoryStatus.operationType` | string | Yes | Current operation. One of `INVENTORY`, `ACCESS`, `NONE`. |
| `heartbeat.inventoryStatus.totalTagsRead` | integer | Yes | Cumulative tag reads since operation start. |
| `heartbeat.inventoryStatus.readRate` | integer | Yes | Current read rate in tags per second. |
| `heartbeat.networkStatus` | object | No | Network connectivity data. Included when network reporting is enabled. |
| `heartbeat.networkStatus.connectionType` | string | Yes | Connection medium. One of `WIFI`, `ETHERNET`, `BLUETOOTH`. |
| `heartbeat.networkStatus.ssid` | string | Yes | SSID of the connected Wi-Fi network (empty if not Wi-Fi). |
| `heartbeat.networkStatus.signalStrength` | integer | Yes | Signal strength in dBm. |
| `heartbeat.networkStatus.ipAddress` | string | Yes | Device IP address. |

```json
  "heartbeat": {
    "uptimeSeconds": 86420,
    "temperature": 34,
    "batteryStatus": {
      "stateOfHealth": "GOOD",
      "chargePercentage": 78,
      "chargeStatus": 0,
      "voltage": 3850,
      "temperature": 31
    },
    "inventoryStatus": {
      "radioActivity": "ACTIVE",
      "operationType": "INVENTORY",
      "totalTagsRead": 14523,
      "readRate": 320
    },
    "networkStatus": {
      "connectionType": "WIFI",
      "ssid": "WAREHOUSE-5G",

      "ipAddress": "10.0.12.87"
    }
  }
}
```
