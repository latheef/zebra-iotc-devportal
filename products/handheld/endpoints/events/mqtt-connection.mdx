# MQTT Connection Event

The `mqttConnEVT` event is published when the device's MQTT connection state changes. This includes initial connections, disconnections, reconnection attempts, and connection failures across all endpoint types (MGMT, CTRL, DATA, MDM).

## Endpoint Details

| Property | Value |
|----------|-------|
| **Event Name** | `mqttConnEVT` |
| **Interface** | MGMT |
| **Direction** | SUB (subscribe to receive) |
| **Topic** | `{tenantId}/MGMT/clients/event/{deviceSerial}` |
| **Content Type** | `application/json` |

## Event Payload (SUB)

Each connection event identifies the affected endpoint, the broker address, the new connection status, and optionally the reason for the state change and the reconnection attempt count.

```json
{
  "event": "mqttConnEVT",
  "deviceSerial": "string",
  "timestamp": "2026-04-14T12:00:00.000Z",
  "apiVersion": "string",
  "connection": {
    "status": "CONNECTED",
    "endpointName": "string",
    "endpointType": "MGMT",
    "brokerHost": "string",
    "brokerPort": 8883,
    "reason": "CLEAN_CONNECT",
    "reconnectAttempt": 0
  }
}
```

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event` | string | Yes | Constant value `"mqttConnEVT"`. |
| `deviceSerial` | string | Yes | Serial number of the reporting device. |
| `timestamp` | string (date-time) | Yes | ISO 8601 timestamp of the event. |
| `apiVersion` | string | Yes | API version identifier. |
| `connection` | object | Yes | Connection state detail container. |
| `connection.status` | string | Yes | New connection state. One of `CONNECTED`, `DISCONNECTED`, `RECONNECTING`, `CONNECTION_FAILED`. |
| `connection.endpointName` | string | Yes | Logical name of the MQTT endpoint. |
| `connection.endpointType` | string | Yes | Endpoint interface type. One of `MGMT`, `CTRL`, `DATA`, `MDM`. |
| `connection.brokerHost` | string | Yes | Hostname or IP address of the MQTT broker. |
| `connection.brokerPort` | integer | Yes | Port number of the MQTT broker. |
| `connection.reason` | string | No | Reason for the state change. One of `CLEAN_CONNECT`, `BROKER_UNAVAILABLE`, `KEEPALIVE_TIMEOUT`, `NETWORK_CHANGE`. |
| `connection.reconnectAttempt` | integer | No | Reconnection attempt counter. `0` for initial connection. |

## Example

```json
{
  "event": "mqttConnEVT",
  "deviceSerial": "RFD40-212735201D0053",
  "timestamp": "2026-04-14T08:01:15.000Z",
  "apiVersion": "1.0",
  "connection": {
    "status": "CONNECTED",
    "endpointName": "production-mgmt",
    "endpointType": "MGMT",
    "brokerHost": "mqtt.example.com",
    "brokerPort": 8883,
    "reason": "CLEAN_CONNECT",
    "reconnectAttempt": 0
  }
}
```
