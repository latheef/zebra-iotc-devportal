# dataEVT Payload Schema

The `dataEVT` payload is published by the RFD40/RFD90 each time one or more tags are read during an RFID operation. This reference documents every field, its type, possible values, and the conditions under which it appears.

## Topic Pattern

```
{tenantId}/DATA/clients/{channel}/{deviceSerial}
```

Where `{channel}` is `data1event` or `data2event` depending on `config_events` channel assignment.

## QoS Level

`dataEVT` messages are published at **QoS 0** by default to maximize throughput. This means delivery is best-effort with no broker acknowledgment.

## Complete Field Table

| Field | Type | Always Present | Description |
|---|---|---|---|
| `event_type` | string | Yes | Always `"dataEVT"` |
| `device_serial` | string | Yes | Serial number of the reader that generated the event |
| `timestamp` | string | Yes | ISO 8601 UTC timestamp of the read: `2026-05-14T10:23:45.123Z` |
| `sequence_id` | integer | Yes | Monotonically increasing sequence number per device session |
| `epc` | string | Yes | Hex-encoded EPC value (e.g., `"3034257BF400B7800004E3A1"`) |
| `epc_length` | integer | Yes | Length of the EPC in bits (typically 96 or 128) |
| `rssi` | number | RSSI/Locationing modes | Received Signal Strength Indicator in dBm (e.g., `-42.5`) |
| `peak_rssi` | number | RSSI/Locationing modes | Peak RSSI observed across antenna cycles |
| `tid` | string | TID mode only | Hex-encoded Tag Identifier from the TID memory bank |
| `phase` | number | Locationing mode only | Phase angle in degrees (0–360) |
| `distance` | number | Locationing mode only | Relative distance indicator (arbitrary units) |
| `channel_index` | integer | Yes | RF channel index used for the read |
| `antenna_id` | integer | Yes | Antenna port identifier (typically `0` for handheld) |
| `seen_count` | integer | Yes | Number of times this tag was observed in the current batch |
| `first_seen` | string | Yes | ISO 8601 timestamp of the first observation in this batch |
| `last_seen` | string | Yes | ISO 8601 timestamp of the most recent observation in this batch |
| `operating_mode` | string | Yes | Mode active during this read: `SIMPLE_INVENTORY`, `INVENTORY_RSSI`, `INVENTORY_TID`, `TAG_LOCATIONING` |
| `pc_bits` | string | No | Protocol Control bits (hex string), present when available |

## Payload Variations by Operating Mode

### Simple Inventory

```json
{
  "event_type": "dataEVT",
  "device_serial": "RFD401234567890",
  "timestamp": "2026-05-14T10:23:45.123Z",
  "sequence_id": 1042,
  "epc": "3034257BF400B7800004E3A1",
  "epc_length": 96,
  "channel_index": 3,
  "antenna_id": 0,
  "seen_count": 1,
  "first_seen": "2026-05-14T10:23:45.123Z",
  "last_seen": "2026-05-14T10:23:45.123Z",
  "operating_mode": "SIMPLE_INVENTORY"
}
```

```json
  "device_serial": "RFD401234567890",
  "timestamp": "2026-05-14T10:23:45.456Z",
  "sequence_id": 1043,
  "epc": "3034257BF400B7800004E3A1",
  "epc_length": 96,
```

```json
  "epc_length": 96,
  "rssi": -35.1,
  "peak_rssi": -32.8,
  "phase": 127.4,

  "seen_count": 5,

  "last_seen": "2026-05-14T10:23:47.012Z",

```

## Timestamp Format

All timestamps use ISO 8601 format in UTC with millisecond precision:

```
YYYY-MM-DDTHH:MM:SS.mmmZ
```

Example: `2026-05-14T10:23:45.123Z`

## Sequence ID Behavior

The `sequence_id` is a monotonically increasing integer that resets when the device reboots or when a new RFID operation starts. Use it to detect gaps in the event stream (missed messages at QoS 0).
