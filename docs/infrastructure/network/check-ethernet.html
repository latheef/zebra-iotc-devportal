How to Check Ethernet Status (Cradle-Connected)

When an RFD40 or RFD90 reader is docked in a charging cradle with an Ethernet connection, the reader can use the cradle's wired network interface. The get_eth command retrieves the current Ethernet status; useful for verifying connectivity before deploying readers in fixed cradle-mounted scenarios, or for diagnosing network issues in docked configurations.

Read-only. Ethernet settings are determined by the cradle hardware and DHCP server. You cannot configure Ethernet parameters directly on the reader. To change IP addressing, modify your DHCP server configuration or cradle network settings.

Publish to:

{tenantId}/MGMT/clients/cmnd/{deviceSerial}

Subscribe for responses on:

{tenantId}/MGMT/clients/resp/{deviceSerial}

Query Ethernet Status​

Request:

{
  "requestId": "req-eth-001",
  "command": "get_eth"
}

Response (connected):

{
  "requestId": "req-eth-001",
  "command": "get_eth",
  "responseCode": 0,
  "responseMessage": "Success",
  "payload": {
    "interfaceStatus": "UP",
    "linkSpeed": 100,
    "duplex": "FULL",
    "macAddress": "00:A0:C9:14:C8:29",
    "ipAddress": "10.1.50.101",
    "subnetMask": "255.255.255.0",
    "gateway": "10.1.50.1",
    "dns": ["10.1.1.10", "10.1.1.11"],
    "dhcpEnabled": true,
    "dhcpLeaseExpiry": "2026-05-15T08:30:00Z"
  }
}

Response (not docked / no Ethernet):

{
  "requestId": "req-eth-001",
  "command": "get_eth",
  "responseCode": 0,
  "responseMessage": "Success",
  "payload": {
    "interfaceStatus": "DOWN",
    "linkSpeed": 0,
    "duplex": "UNKNOWN",
    "macAddress": "00:A0:C9:14:C8:29",
    "ipAddress": "",
    "subnetMask": "",
    "gateway": "",
    "dns": [],
    "dhcpEnabled": false,
    "dhcpLeaseExpiry": ""
  }
}

Field Reference​

FieldTypeDescription
interfaceStatusStringUP if the Ethernet link is active, DOWN if the reader is undocked or the cable is disconnected.
linkSpeedIntegerNegotiated link speed in Mbps (typically 100 or 1000). 0 when the link is down.
duplexStringFULL, HALF, or UNKNOWN.
macAddressStringThe MAC address of the cradle's Ethernet interface as seen by the reader.
ipAddressStringIPv4 address assigned via DHCP (or static, if configured on the cradle). Empty when the link is down.
subnetMaskStringSubnet mask for the assigned IP address.
gatewayStringDefault gateway address.
dnsArrayDNS server addresses provided by DHCP.
dhcpEnabledBooleanWhether the Ethernet interface is using DHCP for address assignment.
dhcpLeaseExpiryStringISO 8601 timestamp of the DHCP lease expiration. Empty if DHCP is not active.

When to Use Ethernet Status Checks​

Cradle-mounted deployments: Readers mounted in cradles at dock doors or workstations can use Ethernet for a more stable, higher-bandwidth connection than Wi-Fi. Verify the link is up before relying on it.

Troubleshooting connectivity: If a docked reader is not reaching the MQTT broker, check Ethernet status first. A DOWN interface means the physical connection (cradle seating, cable, switch port) needs attention.

Monitoring scripts: Periodically query get_eth across your fleet to detect docked readers that have lost Ethernet connectivity; this can indicate cradle hardware issues or network switch problems.

Common Issues​

SymptomLikely CauseAction
