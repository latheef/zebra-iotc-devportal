How to Monitor Battery Lifecycle

Battery management is critical for handheld RFID operations. The RFD40/RFD90 provides battery data through heartbeat events, alerts, and on-demand status checks. This guide covers monitoring strategies, drain pattern analysis, and operational recommendations.

Battery Data Sources​

SourceFieldUpdate Frequency
heartBeatEVTbattery_level, battery_chargingEvery heartbeat interval
alertsBATTERY_LOW, BATTERY_CRITICALOn threshold breach
get_statusbattery_level, battery_charging, battery_healthOn demand

Understanding Battery Levels​

The battery_level field reports percentage remaining (0–100). This is an estimate based on voltage readings and discharge curves; it may fluctuate slightly under varying load conditions.

LevelStatusAction
80–100%Full / near-fullNormal operation
50–80%AdequateMonitor for shift coverage
20–50%LowPlan to dock or swap within 1–2 hours
10–20%WarningBATTERY_LOW alert fires. Swap soon.
0–10%CriticalBATTERY_CRITICAL alert fires. Swap immediately.

Drain Patterns by Operating Mode​

Battery consumption varies significantly by operating mode and configuration:

ModeApproximate Drain RateNotes
Idle (standby)~1% per hourHeartbeats and MQTT keepalive only
Simple Inventory~5–8% per hourDepends on scan frequency and duration
Inventory with RSSI~7–10% per hourAdditional signal processing per tag
Inventory with TID~10–15% per hourMultiple RF exchanges per tag
Tag Locationing~15–25% per hourContinuous RF transmission

These rates assume default antenna power (27.0 dBm) and moderate tag populations. Higher power, more tags, or continuous scanning increases drain.

Tracking Drain Rates​

Monitor the rate of battery decline from heartbeat events to predict when a device will need charging:

const batteryHistory = new Map(); // deviceSerial -> [{timestamp, level}]

function trackBattery(heartbeat) {
  const serial = heartbeat.device_serial;
  if (!batteryHistory.has(serial)) {
    batteryHistory.set(serial, []);
  }

  const history = batteryHistory.get(serial);
  history.push({
    timestamp: new Date(heartbeat.timestamp),
    level: heartbeat.battery_level
  });

  // Keep last 60 data points
  if (history.length > 60) history.shift();

  // Calculate drain rate (%/hour) from last 10 minutes of data
  if (history.length >= 2) {
    const oldest = history[0];
    const newest = history[history.length - 1];
    const hours = (newest.timestamp - oldest.timestamp) / 3600000;
    const drainPerHour = (oldest.level - newest.level) / hours;
    const hoursRemaining = newest.level / drainPerHour;

    return { drainPerHour, hoursRemaining, currentLevel: newest.level };
  }
}

Configuring Battery Alert Thresholds​

Set thresholds appropriate for your operational needs:

{
  "command": "config_events",
  "command_id": "cmd-070",
  "payload": {
    "action": "SET",
    "battery_warn_threshold": 25,
    "battery_critical_threshold": 10
  }
}

For operations requiring guaranteed battery life (e.g., a 4-hour inventory count), set the warning threshold high enough to provide time for a swap:

Shift LengthRecommended WarningRecommended Critical
2 hours15%5%
4 hours25%10%
8 hours30%15%

Battery Health Monitoring​

The battery_health field in get_status indicates long-term battery condition:

  • GOOD: Battery holds charge within spec. Normal operation.
  • DEGRADED: Reduced capacity compared to new. Battery may not last a full shift. Plan replacement.
  • REPLACE: Battery is significantly degraded. Replace before it fails unexpectedly.

Track battery_health across the fleet to schedule preventive battery replacements before they impact operations.

Operational Recommendations​

  1. Start shifts fully charged: dock all devices at shift end
  2. Monitor drain rates: devices draining faster than expected may have degraded batteries or inefficient configurations
  3. Reduce power when possible: lower antenna_power in small-area scanning reduces battery consumption
  4. Use S1 session: avoids re-reading the same tags repeatedly, reducing RF time
  5. Increase heartbeat interval: in battery-sensitive deployments, use 300-second heartbeats instead of 30-second
  6. Rotate batteries: in multi-shift operations, keep a rotation of charged batteries available

Warning: Avoid letting batteries discharge to 0% regularly. Deep discharge cycles reduce long-term battery capacity. Configure BATTERY_CRITICAL thresholds to catch devices before full depletion.
