How to Configure Event Reporting

This guide covers configuring which events the RFD40/RFD90 publishes, their intervals, thresholds, and data channel assignments using the config_events MQTT command.

Prerequisites​

  • Device connected and online
  • MQTT client subscribed to the control response topic

Reading the Current Event Configuration​

Publish to:

{tenantId}/CTRL/clients/cmnd/{deviceSerial}
{
  "command": "config_events",
  "command_id": "cmd-050",
  "payload": {
    "action": "GET"
  }
}

Response on:

{tenantId}/CTRL/clients/resp/{deviceSerial}
{
  "command": "config_events",
  "command_id": "cmd-050",
  "status": "OK",
  "payload": {
    "heartbeat_enabled": true,
    "heartbeat_interval_sec": 60,
    "alerts_enabled": true,
    "alert_short_enabled": false,
    "exception_enabled": true,
    "mqtt_conn_enabled": true,
    "data_channel_mode": "SINGLE",
    "battery_warn_threshold": 20,
    "battery_critical_threshold": 10,
    "temperature_threshold_c": 45
  }
}

Enable or Disable Specific Events​

Disable Alert Short Events​

{
  "command": "config_events",
  "command_id": "cmd-051",
  "payload": {
    "action": "SET",
    "alert_short_enabled": false
  }
}

Enable All Events​

{
  "command": "config_events",
  "command_id": "cmd-052",
  "payload": {
    "action": "SET",
    "heartbeat_enabled": true,
    "alerts_enabled": true,
    "alert_short_enabled": true,
    "exception_enabled": true,
    "mqtt_conn_enabled": true
  }
}

Configure Heartbeat Interval​

The heartbeat interval controls how frequently heartBeatEVT events are published. Shorter intervals give faster health visibility but increase battery and bandwidth consumption.

{
  "command": "config_events",
  "command_id": "cmd-053",
  "payload": {
    "action": "SET",
    "heartbeat_interval_sec": 30
  }
}
IntervalUse CaseBattery Impact
15 secondsReal-time fleet dashboardsHigh
30 secondsActive monitoringModerate
60 secondsStandard monitoring (default)Low
300 secondsBattery-conservative environmentsMinimal

Configure Alert Thresholds​

Set the metric thresholds that trigger alert events:

{
  "command": "config_events",
  "command_id": "cmd-054",
  "payload": {
    "action": "SET",
    "battery_warn_threshold": 25,
    "battery_critical_threshold": 10,
    "temperature_threshold_c": 40
  }
}

Threshold Fields​

FieldTypeDefaultDescription
battery_warn_thresholdinteger20Battery % that triggers a WARNING alert
battery_critical_thresholdinteger10Battery % that triggers a CRITICAL alert
temperature_threshold_cnumber45Temperature (°C) that triggers a temperature alert

Configure Data Channel Assignment​

Control how dataEVT messages are distributed across the two data channels:

{
  "command": "config_events",
  "command_id": "cmd-055",
  "payload": {
    "action": "SET",
    "data_channel_mode": "SPLIT"
  }
}
ModeBehavior
SINGLEAll data to data1event (default)
SPLITData distributed across both channels
DUPLICATEData sent to both channels (testing)

Complete Configuration Example​

Apply a full event configuration in a single command:

{
  "command": "config_events",
  "command_id": "cmd-056",
  "payload": {
    "action": "SET",
    "heartbeat_enabled": true,
    "heartbeat_interval_sec": 30,
    "alerts_enabled": true,
    "alert_short_enabled": true,
    "exception_enabled": true,
    "mqtt_conn_enabled": true,
    "data_channel_mode": "SINGLE",
    "battery_warn_threshold": 25,
    "battery_critical_threshold": 10,
    "temperature_threshold_c": 42
  }
}

Verify the Configuration​

Read back the configuration to confirm changes:

{
  "command": "config_events",
  "command_id": "cmd-057",
  "payload": {
    "action": "GET"
  }
}

Error Handling​

Error CodeCauseResolution
INVALID_INTERVALHeartbeat interval outside valid rangeUse 15–600 seconds
INVALID_THRESHOLDThreshold value out of rangeCheck valid ranges for each metric
INVALID_MODEUnrecognized data channel modeUse SINGLE, SPLIT, or DUPLICATE

Note: Event configuration changes take effect immediately. There is no need to restart the device or stop active operations. However, changing the data channel mode while an RFID operation is running may cause a brief gap in data delivery during the transition.
