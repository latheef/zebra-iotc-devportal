Appendix A: Response Codes

Zebra IoT Connector: MQTT API V1.1

Every command response includes a response.code integer field. Find your code below for meaning, typical causes, and recommended action.

Response Code Table​

CodeDescriptionTypical CommandsCauseRecommended Action
0SuccessAll commandsCommand executed successfullyNone required
1Command payload acceptedset_config, config_endpoint, config_events, set_osCommand queued for asynchronous processingMonitor response topic for final result
2Invalid payloadAll commandsMalformed JSON, missing required fields, or invalid field valuesValidate the payload against the command's schema
3Unable to retrieve infoAll getter commandsDevice in transitional state, rebooting, or requested data unavailableRetry after 2–3 seconds
6Region not configuredcontrol_operationRFID operation attempted without a configured regulatory regionSet the region via 123RFID Desktop
10Config already existsconfig_endpoint, set_wifi, install_certificateADD operation for an endpoint, Wi-Fi profile, or certificate that already existsUse MODIFY operation, or DELETE first then ADD
11Config does not existconfig_endpoint, delete_certificateMODIFY or DELETE for a config that does not existVerify the config name with the corresponding GET command
12Operation already in progresscontrol_operationSTART command sent while an operation is already activeSTOP the current operation first
15SSID not foundset_wifi, delete_wifi_profileWi-Fi SSID not detected or not in stored profilesVerify SSID spelling; confirm AP is broadcasting
17SSID missingset_wifiWi-Fi configuration payload lacks the ssid fieldInclude the ssid field
18Certificate parse errorinstall_certificateInvalid PEM format or corrupted certificate dataVerify PEM encoding with proper BEGIN/END headers
19Download failedset_osReader could not connect to or download from the firmware URLVerify URL accessibility from the reader's network
20Firmware validation failedset_osDownloaded firmware package integrity check failedVerify the firmware package is not corrupted
21Insufficient batteryset_osBattery too low for firmware update (typically < 30%)Charge the reader or connect to USB power
22Operation in progresscontrol_operation, reboot, set_operating_modeA conflicting operation prevents executionWait for the current operation to complete
23Radio not connectedcontrol_operation, set_operating_modeRFID radio is not connected or initializedCheck Bluetooth pairing; verify radio status via get_status
24Invalid operating modeset_operating_modeOperating mode parameter combination is invalidReview the mode configuration against supported combinations
28Feature not supportedset_operating_modeRequested feature requires a newer firmware versionUpdate firmware via set_os

Response Code Ranges​

RangeCategoryDescription
0SuccessCommand completed successfully
1AcceptedCommand accepted for async processing
2–5Input errorsPayload validation failures
6–9Precondition errorsDevice state prerequisites not met
10–14Configuration conflictsDuplicate or missing configuration
15–18Resource errorsResource not found or invalid
19–21Firmware errorsFirmware update failures
22–28Operational errorsRuntime conflicts and feature limitations

Note on dual-meaning codes: Codes 11 and 22 apply to different domains. Code 11 means "inventory in progress" in the Control domain and "config does not exist" in the Management domain. Code 22 means "operation in progress" for reboot/set_os and "advanced configuration not set" for control_operation. The command context determines which interpretation applies.
