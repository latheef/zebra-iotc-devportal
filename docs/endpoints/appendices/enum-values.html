Appendix B: Enum Values

Zebra IoT Connector: MQTT API V1.1

All enumeration values used across the MQTT API are listed below, grouped by field name. Each entry includes the commands or events that use it.

controlType: control_operation​

ValueDescription
RFIDRFID radio operations
SCANNERBarcode scanner operations (if supported by hardware)

operation: control_operation​

ValueDescription
STARTStart the operation
STOPStop the operation

securityType: set_wifi, get_wifi​

ValueDescription
WPA2_PERSONALWPA2 with pre-shared key
WPA3_PERSONALWPA3 with SAE authentication
WPA2_ENTERPRISEWPA2 with 802.1X / EAP
WPA3_ENTERPRISEWPA3 with 802.1X / EAP
OPENNo authentication

eapMethod: set_wifi, get_wifi​

ValueDescription
PEAPProtected EAP
TLSEAP-TLS (certificate-based)
TTLSEAP-TTLS (tunneled)

phase2Method: set_wifi​

ValueDescription
MSCHAPV2MS-CHAPv2 inner authentication
GTCGeneric Token Card
PAPPassword Authentication Protocol

wifiOperation: set_wifi​

ValueDescription
ADDCreate a new Wi-Fi profile
MODIFYUpdate an existing Wi-Fi profile

endpointOperation: config_endpoint​

ValueDescription
ADDCreate a new endpoint
MODIFYUpdate an existing endpoint
DELETERemove an endpoint

endpointType: config_endpoint, get_endpoint_config​

ValueDescription
MGMTManagement interface
CTRLControl interface
DATATag data interface
MDMMobile device management

connectionStatus: get_endpoint_config, mqttConnEVT​

ValueDescription
CONNECTEDConnected to the broker
DISCONNECTEDDisconnected from the broker
CONNECTINGConnection in progress
RECONNECTINGReconnection attempt in progress
CONNECTION_FAILEDConnection attempt failed

certificateType: install_certificate, get_installed_certificate​

ValueDescription
mqttMQTT broker TLS certificate
wifiWi-Fi EAP-TLS certificate
clientClient certificate for mutual TLS

linkStatus: get_eth​

ValueDescription
UPCable connected and link established
DOWNNo cable or no link

powerSource: get_status​

ValueDescription
BATTERYRunning on battery power
USBConnected to USB power
CRADLEDocked in a cradle

radioActivity: get_status, heartBeatEVT​

ValueDescription
ACTIVERFID radio is actively performing operations
INACTIVERFID radio is idle

radioConnection: get_status​

ValueDescription
CONNECTEDRFID radio is connected and ready
DISCONNECTEDRFID radio is not connected

stateOfHealth: get_status, heartBeatEVT​

ValueDescription
GOODBattery health is good
DEGRADEDBattery health has degraded; consider replacement
REPLACEBattery should be replaced

filterType: set_post_filter​

ValueDescription
PREFIXMatch tags whose EPC starts with the pattern
SUFFIXMatch tags whose EPC ends with the pattern
REGEXMatch tags whose EPC matches the regular expression
RSSIFilter by received signal strength
WHITELISTAllow only tags in the list
BLACKLISTExclude tags in the list

filterAction: set_post_filter​

ValueDescription
INCLUDEInclude matching tags in data events
EXCLUDEExclude matching tags from data events

alertType: alerts, alert_short​

ValueDescription
TEMPERATURE_HIGHInternal temperature exceeded threshold
BATTERY_LOWBattery below low threshold
BATTERY_CRITICALBattery below critical threshold
BATTERY_HEALTH_DEGRADEDBattery health is degraded
RADIO_ERRORRFID radio error detected

severity: alerts, alert_short​

ValueDescription
INFOInformational
WARNINGWarning condition
CRITICALCritical condition requiring immediate action

exceptionType: exceptionEVT​

ValueDescription
RADIO_COMMUNICATION_ERRORRFID radio communication failure
RADIO_INITIALIZATION_ERRORRFID radio failed to initialize
BLUETOOTH_DISCONNECTIONBluetooth connection lost
NETWORK_ERRORWi-Fi or Ethernet connectivity failure
MQTT_PUBLISH_ERRORMQTT publish failure
FIRMWARE_ERRORInternal firmware error
MEMORY_ERRORMemory allocation failure
HARDWARE_ERRORHardware fault detected

mqttConnStatus: mqttConnEVT​

ValueDescription
CONNECTEDConnected to the broker
DISCONNECTEDDisconnected from the broker
RECONNECTINGReconnection in progress
CONNECTION_FAILEDConnection attempt failed

connReason: mqttConnEVT​

ValueDescription
CLEAN_CONNECTInitial clean connection
BROKER_UNAVAILABLEBroker became unreachable
KEEPALIVE_TIMEOUTMQTT keepalive timeout exceeded
NETWORK_CHANGENetwork interface changed (e.g., Wi-Fi to Ethernet)
