MQTT API Reference Overview

Complete inventory of MQTT 3.1.1 endpoints for managing and operating Zebra RFD40/RFD90 handheld RFID reader sleds through the IoT Connector.

Protocol​

All communication uses MQTT 3.1.1 with JSON-encoded payloads. The reader connects to a configured MQTT broker and subscribes to command topics. Applications publish commands and subscribe to response and event topics.

Topic Pattern​

{tenantId}/{interface}/clients/{channel}/{deviceSerial}
SegmentDescription
tenantIdTenant identifier assigned during provisioning
interfaceMGMT, CTRL, or DATA
channelcmnd (command), resp (response), event, data1event, or data2event
deviceSerialDevice serial number (e.g., 24042AR50033)

Interfaces​

InterfacePurposeChannels
MGMTDevice management; configuration, networking, firmware, certificatescmnd, resp, event
CTRLRFID operations; operating modes, filters, start/stopcmnd, resp
DATATag data; read events streamed from the readerdata1event, data2event

Channel Directions​

ChannelDirectionDescription
cmndApplication → ReaderCommands sent to the device
respReader → ApplicationResponses to commands
eventReader → ApplicationAsynchronous events (heartbeats, alerts)
data1event / data2eventReader → ApplicationTag read data

Payload Format​

All payloads are JSON objects. Every command includes a command field and a requestId field. Every response includes a command field, requestId, and a statusCode.

Request Structure​

{
  "command": "<command_name>",
  "requestId": "<unique_id>",
  ...
}

Response Structure​

{
  "command": "<command_name>",
  "requestId": "<unique_id>",
  "statusCode": 0,
  "statusMessage": "Success",
  ...
}

Standard Status Codes​

CodeNameDescription
0SuccessCommand executed successfully
1GeneralErrorAn unspecified error occurred
2InvalidCommandThe command name is not recognized
3InvalidParameterA parameter value is out of range or malformed
4InvalidRequestIdThe requestId field is missing or empty
5DeviceBusyThe device is processing another command
10OperationFailedThe command was understood but execution failed
15NotFoundThe requested resource does not exist
16ResourceInUseThe resource cannot be modified because it is active

Field Table Format​

Throughout this reference, field tables use the following columns:

ColumnDescription
FieldJSON path using dot notation (e.g., ntp.status)
Typestring, integer, boolean, number, array, object
RequiredWhether the field must be present in the request
DescriptionMeaning and constraints

Events: 4 Event Types​

#EventDescription

Functional Groups​

  1. Device Identity: get_version
  2. Device Health: get_status, heartBeatEVT, alerts, alert_short
  3. Networking: get_eth, get_wifi, set_wifi, delete_wifi_profile
  4. Security: get_installed_certificate, install_certificate, delete_certificate
  5. Endpoint Management: get_endpoint_config, config_endpoint
  6. Configuration: get_config, set_config, config_events
  7. RFID Operations: get_operating_mode, set_operating_mode, control_operation, get_post_filter, set_post_filter, dataEVT
  8. Firmware & Lifecycle: set_os, reboot, mqttConnEVT
