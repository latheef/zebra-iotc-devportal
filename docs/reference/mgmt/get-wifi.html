get_wifi

Queries the Wi-Fi profiles configured on the device, including connection status, security type, and IP configuration for each profile.

Topic​

Topic
Request{tenantId}/MGMT/clients/cmnd/{deviceSerial}
Response{tenantId}/MGMT/clients/resp/{deviceSerial}

Direction​

Application → Reader → Application

Request Payload​

{
  "command": "get_wifi",
  "requestId": "5"
}
FieldTypeRequiredDescription
commandstringYesMust be "get_wifi"
requestIdstringYesUnique identifier to correlate the response

Response Payload​

  "requestId": "5",
  "statusCode": 0,
  "statusMessage": "Success",
  "wifiProfiles": {
    "wifiConfig": [

          "isPreferred": true,
          "securityType": "WPA2Personal",
          "signalStrength": -45
        },
        "ipv4Configuration": {
          "ipAddress": "10.0.5.22",

          "enableDhcp": true,

      },
      {
        "accessPoint": {

FieldTypeDescription
commandstringEcho of the command name
requestIdstringEcho of the request identifier
statusCodeinteger0 for success
statusMessagestringHuman-readable status
wifiProfiles.wifiConfigarrayList of configured Wi-Fi profiles
wifiProfiles.wifiConfig[].accessPoint.essidstringNetwork SSID
wifiProfiles.wifiConfig[].accessPoint.statusstringCONNECTED, SAVED, CONNECTING, DISCONNECTED
wifiProfiles.wifiConfig[].accessPoint.isPreferredbooleanWhether this is the preferred network
wifiProfiles.wifiConfig[].accessPoint.securityTypestringWPA2Personal, WPA3Personal, WPA2Enterprise, WPA3Enterprise, OPEN
wifiProfiles.wifiConfig[].accessPoint.signalStrengthintegerRSSI in dBm. 0 when not connected
wifiProfiles.wifiConfig[].ipv4Configuration.ipAddressstringAssigned IPv4 address (empty if not connected)
wifiProfiles.wifiConfig[].ipv4Configuration.subnetMaskstringSubnet mask
wifiProfiles.wifiConfig[].ipv4Configuration.enableDhcpbooleantrue if address is obtained via DHCP
wifiProfiles.wifiConfig[].ipv4Configuration.gatewaystringDefault gateway address

Error Responses​

CodeNameDescription
1GeneralErrorUnexpected internal error
4InvalidRequestIdrequestId is missing or empty

Notes​

  • The device supports a maximum of 10 Wi-Fi profiles.
  • Only one profile can have status: "CONNECTED" at a time.
  • signalStrength is reported only for the currently connected network. Saved profiles return 0.
  • Enterprise profiles do not expose credential details in the response for security reasons.
