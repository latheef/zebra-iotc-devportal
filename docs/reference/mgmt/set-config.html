set_config

Applies a partial or full configuration document to the device. This is the most comprehensive configuration endpoint, capable of updating network, MQTT, event, and RFID settings in a single command.

Topic​

Topic
Request{tenantId}/MGMT/clients/cmnd/{deviceSerial}
Response{tenantId}/MGMT/clients/resp/{deviceSerial}

Direction​

Application → Reader → Application

Request Payload​

The request accepts any subset of the configuration document. Only the fields included are updated; omitted fields retain their current values.

Partial Update Example​

{
  "command": "set_config",
  "requestId": "17",
  "config": {
    "events": {
      "heartbeat": {
        "intervalSeconds": 600
      }
    },
    "rfid": {
      "operatingMode": {
        "transmitPowerIndex": 240,
        "session": "S1"
      }
    }
  }
}

Full Configuration Example​

{
  "command": "set_config",
  "requestId": "18",
  "config": {
    "network": {
      "ethernet": {
        "ipv4Configuration": {
          "enableDhcp": false,
          "ipAddress": "192.168.1.100",
          "subnetMask": "255.255.255.0",
          "gateway": "192.168.1.1",
          "dns1": "8.8.8.8"
        }
      }
    },
    "mqtt": {
      "activeEndpoint": "Production-Broker"
    },
    "events": {
      "heartbeat": {
        "enabled": true,
        "intervalSeconds": 300
      },
      "alerts": {
        "enabled": true,
        "batteryLow": {
          "enabled": true,
          "thresholdPercent": 15
        },
        "temperatureHigh": {
          "enabled": true,
          "thresholdCelsius": 45
        }
      }
    },
    "rfid": {
      "operatingMode": {
        "mode": "INVENTORY",
        "transmitPowerIndex": 270,
        "session": "S0",
        "tagPopulation": 100
      }
    }
  }
}
FieldTypeRequiredDescription
commandstringYesMust be "set_config"
requestIdstringYesUnique identifier to correlate the response
configobjectYesConfiguration document (partial or full)
config.networkobjectNoNetwork settings (Ethernet, Wi-Fi)
config.network.ethernet.ipv4Configuration.enableDhcpbooleanNotrue for DHCP, false for static IP
config.network.ethernet.ipv4Configuration.ipAddressstringConditionalRequired when enableDhcp is false
config.network.ethernet.ipv4Configuration.subnetMaskstringConditionalRequired when enableDhcp is false
config.network.ethernet.ipv4Configuration.gatewaystringConditionalRequired when enableDhcp is false
config.mqtt.activeEndpointstringNoName of the endpoint to activate
config.eventsobjectNoEvent and alert configuration
config.rfid.operatingMode.modestringNoRFID mode (see set_operating_mode for options)
config.rfid.operatingMode.transmitPowerIndexintegerNoTransmit power in tenths of dBm (0–300)
config.rfid.operatingMode.sessionstringNoRFID session: S0, S1, S2, S3
config.rfid.operatingMode.tagPopulationintegerNoEstimated tag population (1–10000)

Response Payload​

{
  "command": "set_config",
  "requestId": "17",
  "statusCode": 0,
  "statusMessage": "Success",
  "appliedSections": ["events", "rfid"]
}
FieldTypeDescription
commandstringEcho of the command name
requestIdstringEcho of the request identifier
statusCodeinteger0 for success
statusMessagestringHuman-readable status
appliedSectionsarrayList of configuration sections that were updated

Error Responses​

CodeNameDescription
1GeneralErrorUnexpected internal error
3InvalidParameterA parameter value is out of range or malformed
4InvalidRequestIdrequestId is missing or empty
5DeviceBusyThe device is currently performing an RFID operation; stop it first
10OperationFailedOne or more configuration sections could not be applied

Notes​

  • set_config applies changes atomically per section. If one section fails, other sections may still be applied. Check appliedSections in the response.
  • Changing mqtt.activeEndpoint causes the device to disconnect from the current broker and reconnect to the new one. Commands in flight during the switch are lost.
  • RFID configuration changes (config.rfid) require that no RFID operation is in progress. Send control_operation with operationType: "STOP" first.
  • The config.deviceInfo section is read-only and is ignored if included.
  • Network changes may cause the device to become temporarily unreachable if the MQTT broker connection depends on the network being reconfigured.
  • Configuration changes persist across reboots.

Warning: Changing network or MQTT settings can make the device unreachable. Ensure you have a recovery path (e.g., physical access or a secondary management interface) before modifying connectivity settings remotely.
