dataEVT

Tag and barcode read data events published by the reader during an active RFID inventory operation. Each event contains tag data and optionally barcode data with EPC, RSSI, antenna, timestamp, and access results.

Topic​

Topic
data1event{tenantId}/DATA/clients/data1event/{deviceSerial}
data2event{tenantId}/DATA/clients/data2event/{deviceSerial}

Direction​

Reader → Application

Payload​

FieldTypeDescription
typestringOperating mode that produced the data (see enum below)
timestampstringEvent publish timestamp in ISO 8601 format
dataobjectTag and barcode data container
data.tagDataarrayArray of tag read records
data.barcodeDataarrayArray of barcode scan records (present when barcodes are scanned)

type Enum​

ValueDescription
FAST_READHigh-speed read mode
CYCLE_COUNTCycle count inventory mode
DENSE_READERSDense reader environment mode
OPTIMAL_BATTERYBattery-optimized mode
BALANCED_PERFORMANCEBalanced speed/battery mode
ADVANCEDAdvanced custom configuration mode

Tag Data Fields (data.tagData[])​

FieldTypeDescription
eventNumnumberEvent sequence number (required)
formatstringEPC encoding format, typically "epc" (required)
EPCidstringEPC identifier — unique identifier encoded on the tag (hex string)
EPCstringEPC memory bank content (hex string)
TIDstringTag Identifier memory bank — manufacturer-assigned, immutable (hex string)
USERstringUser memory bank — application-defined data (hex string)
RESERVEDstringReserved memory bank — kill and access passwords (hex string)
PCstringPC bits of the inventoried tag (hex string)
CRCstringCRC bits of the inventoried tag (hex string)
XPCstringXPC bits, if present (hex string)
channelnumberRF channel in MHz when the tag was read (only when reportFilter duration is 0)
peakRssinumberSignal strength in dBm (peak value since last report)
phasenumberRF phase angle in degrees (only when reportFilter duration is 0)
seenCountnumberTimes the tag was inventoried since the previous report (always 1 when reportFilter duration is 0)
firstSeenTimenumberTimestamp in milliseconds since epoch when the tag was first seen
lastSeenTimenumberTimestamp in milliseconds since epoch when the tag was last seen
accessResultsarrayResults of access operations (read/write/lock/kill) as string array
userDefinedstringCustom string value configured in operating mode
MACstringMAC address of the reader that inventoried the tag
HOSTNAMEstringHostname of the reader that inventoried the tag

Barcode Data Fields (data.barcodeData[])​

FieldTypeDescription
symbologystringBarcode symbology: CODE_39
decodedBarcodestringDecoded barcode string value (required)

Notes​

  • The data1event and data2event topics carry the same event type. Two topics are provided for different QoS or subscriber routing in high-throughput scenarios.
  • The fields present in each tag record depend on the tagReportConfig configured via set_operating_mode. Only enabled fields appear in the payload.
  • EPCid is typically the same as EPC — both represent the Electronic Product Code.
  • peakRssi values are negative dBm. Higher values (closer to 0) indicate stronger signal.
  • firstSeenTime and lastSeenTime are milliseconds since Unix epoch (not ISO 8601 strings).
  • channel and phase are only included when reportFilter duration is set to 0 (per-read reporting).
  • High tag populations can generate significant data volume. Use seenCountThreshold and post-filters to manage throughput.
