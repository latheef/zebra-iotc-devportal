  • 17.3 Exception Event Codes

Exception Event Codes

📕 REFERENCE: Event codes published in exceptionEVT payloads on the device event topic.

Overview​

The IoT Connector publishes exception events when the device encounters abnormal conditions that do not occur as direct responses to commands. These events are delivered asynchronously on the device event topic:

zebra/iotc/event/{deviceId}

Exception events use the exceptionEVT event type and contain a structured payload with an event code, severity, and descriptive message.

Exception Event Payload Structure​

{
  "eventType": "exceptionEVT",
  "eventCode": 1001,
  "severity": "WARNING",
  "message": "Battery level below 20%",
  "timestamp": "2026-05-14T10:30:00.000Z",
  "deviceId": "RFD40-123456"
}
FieldTypeDescription
eventTypeStringAlways "exceptionEVT" for exception events.
eventCodeIntegerNumeric code identifying the specific exception condition.
severityStringOne of INFO, WARNING, ERROR, or CRITICAL.
messageStringHuman-readable description of the exception.
timestampStringISO 8601 timestamp when the exception occurred.
deviceIdStringThe unique identifier of the device that raised the exception.

Severity Levels​

SeverityMeaning
INFOInformational event; no action required.
WARNINGPotential issue that may require attention if it persists.
ERRORAn error condition that impacts device functionality.
CRITICALSevere condition requiring immediate attention; device operation may be compromised.

1003INFOBattery ChargingDevice has been connected to a charging source.
1004INFOBattery Fully ChargedBattery has reached full charge.
1005WARNINGBattery Health DegradedBattery health has dropped below 80% of original capacity. Consider replacement.

Connectivity Events (1100–1199)​

CodeSeverityNameDescription
1100ERRORMQTT Connection LostThe connection to the MQTT broker was lost unexpectedly. The device will attempt to reconnect.
1101INFOMQTT ReconnectedThe device has successfully reconnected to the MQTT broker after a disconnection.
1102WARNINGWi-Fi Signal WeakThe Wi-Fi signal strength has dropped below the acceptable threshold (RSSI below -75 dBm).
1103ERRORWi-Fi DisconnectedThe Wi-Fi connection to the access point was lost.

1108INFOBluetooth ConnectedThe device has paired and connected to a host mobile device via Bluetooth.

RFID Operation Events (1200–1299)​

CodeSeverityNameDescription
1200ERRORAntenna FaultAn antenna fault was detected. RFID operations on the affected antenna will not function.
1201WARNINGHigh Tag PopulationThe tag population in the field exceeds the optimal threshold. Read rates may be reduced.
1202ERRORInventory AbortedThe inventory operation was terminated unexpectedly due to an internal error.
1203WARNINGTemperature WarningThe device internal temperature is approaching the operational limit. Transmit power may be automatically reduced.
1204CRITICALTemperature ShutdownThe device has exceeded its safe operating temperature and is suspending RFID operations to cool down.

CodeSeverityNameDescription
1300WARNINGCertificate Expiring SoonA TLS certificate on the device will expire within 30 days.
1301ERRORCertificate ExpiredA TLS certificate on the device has expired. Secure connections using this certificate will fail.

CodeSeverityNameDescription
1400INFOFirmware Update AvailableA new firmware version is available for the device.
1401INFOFirmware Update StartedThe device has begun downloading and applying a firmware update.
1402INFOFirmware Update CompleteThe firmware update was applied successfully. A reboot may be required.
1403ERRORFirmware Update FailedThe firmware update process failed. See error details in the message field.

Subscribing to Exception Events​

Subscribe to the device event topic to receive exception events alongside other device events (tag reads, heartbeats, etc.):

zebra/iotc/event/{deviceId}

Filter for exception events by checking the eventType field:

def on_message(client, userdata, msg):
    payload = json.loads(msg.payload.decode())

    if payload.get("eventType") == "exceptionEVT":
        code = payload["eventCode"]
        severity = payload["severity"]
        message = payload["message"]
        handle_exception(code, severity, message)

Warning: Exception events are published on a best-effort basis. If the device loses connectivity, exception events that occur during the disconnection period are not guaranteed to be delivered retroactively. Design your monitoring to detect missing heartbeats as an additional signal of device issues.

Recommended Actions by Severity​

SeverityRecommended Action
INFOLog for audit trail. No immediate action required.
WARNINGLog and monitor. Investigate if the condition persists or occurs frequently.
ERRORAlert operations team. Investigate root cause and take corrective action.
CRITICALImmediate response required. Device operation may be degraded or halted.
