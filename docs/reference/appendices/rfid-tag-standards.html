Supported RFID Tag Types & Standards

📕 REFERENCE: EPC Gen2 compliance, memory bank structure, tag populations, and read/write capabilities.

Supported Standard​

The RFD40 and RFD90 reader sleds support EPC Gen2 Version 2 (ISO 18000-63) UHF RFID tags. This is the dominant global standard for passive UHF RFID operating in the 860–960 MHz frequency range.

SpecificationDetails
Air Interface ProtocolEPC Global Class 1 Generation 2 (Gen2v2)
ISO StandardISO 18000-63
Frequency Range860–960 MHz (region-specific sub-bands)
Tag TypePassive (backscatter)
CommunicationReader Talks First (RTF)

Tag Memory Banks​

Every EPC Gen2 tag has four memory banks. The IoT Connector can read from and write to these banks using the appropriate commands.

Bank 0: Reserved Memory​

AttributeDetails
Size64 bits (8 bytes)
ContentsKill password (32 bits) + Access password (32 bits)
DefaultAll zeros (no password set)
Read AccessRequires access password if locked
Write AccessRequires access password if locked
Use CaseSecurity; set passwords to protect tag data or permanently disable tags

Bank 1: EPC Memory​

AttributeDetails
SizeVariable, typically 96 or 128 bits for the EPC; includes CRC-16 and Protocol Control (PC) bits
ContentsCRC-16 (16 bits) + PC bits (16 bits) + EPC (typically 96 bits)
DefaultManufacturer-assigned or blank
Read AccessOpen by default
Write AccessOpen by default (can be locked)
Use CasePrimary tag identifier; used for inventory and item-level tracking

Bank 2: TID Memory (Tag Identifier)​

AttributeDetails
SizeMinimum 64 bits; varies by tag IC
ContentsTag IC manufacturer code, model number, and unique serial number
Read AccessAlways readable
Write AccessPermanently locked (read-only)
Use CaseTag authentication, anti-counterfeiting, and chip identification. The TID is factory-programmed and cannot be changed.

Bank 3: User Memory​

AttributeDetails
Size0 to 65,535 bits (varies by tag IC; many tags have 0, 128, 256, or 512 bits)
ContentsApplication-defined data
DefaultEmpty or all zeros
Read AccessOpen by default
Write AccessOpen by default (can be locked)
Use CaseStoring additional data beyond the EPC, such as serial numbers, maintenance records, or sensor calibration data

Note: Not all tags have User memory. Low-cost tags often omit Bank 3 entirely. Verify the tag's datasheet before attempting User memory operations.

Tag Populations and Performance​

The reader sled's inventory performance depends on the number of tags in the RF field (tag population).

Tag PopulationExpected PerformanceRecommended Session
1–50 tagsFull read in under 1 secondSession 0, Target A
50–200 tagsFull read in 1–3 secondsSession 1, Target AB
200–500 tagsFull read in 3–10 secondsSession 1, Target AB
500–1,000 tagsFull read in 10–30 secondsSession 2, Target AB
1,000+ tagsMay require multiple passesSession 2, Target AB, with tuned RF mode

Tip: For large tag populations, use Session 1 or 2 with Target AB to ensure each tag is singulated exactly once per inventory cycle. Session 0 causes every tag to respond on every cycle, which leads to collisions in dense environments.

Read and Write Operations​

Reading Tags​

The read_tag command reads data from a specific memory bank. You must identify the target tag (typically by EPC), specify the memory bank, word offset, and word count.

ParameterDescription
epcEPC of the target tag (hex string)
memoryBankBank to read: "RESERVED", "EPC", "TID", "USER"
wordOffsetStarting word (16-bit) offset
wordCountNumber of words to read
accessPasswordAccess password (hex, required if bank is locked)

Writing Tags​

The write_tag command writes data to a memory bank. Write operations are slower than reads and require the tag to remain in the RF field for the duration of the write.

ParameterDescription
epcEPC of the target tag (hex string)
memoryBankBank to write: "EPC", "USER", "RESERVED"
wordOffsetStarting word offset for the write
dataData to write (hex string, must align to 16-bit word boundaries)
accessPasswordAccess password (hex, required if bank is locked)

Write considerations:

  • Tag must remain stationary and within range during the write operation
  • Write range is significantly shorter than read range (typically 50–70% of read range)
  • Writing to a locked memory bank without the correct access password returns an error
  • Writing to TID memory (Bank 2) is not possible; it is permanently locked at the factory

Locking Memory Banks​

The lock_tag command changes the access permissions on a memory bank. Lock states include:

Lock StateDescription
UnlockedRead and write access without a password
LockedRead and write access require the access password
PermalockedPermanently locked; cannot be unlocked even with the password
Perma-unlockedPermanently unlocked; cannot be locked

Warning: Permalock and perma-unlock operations are irreversible. Test thoroughly on sample tags before applying to production inventory.

Environmental Considerations​

Tag read performance is affected by the operating environment:

EnvironmentImpactMitigation
Metal surfacesSevere detuning, reduced rangeUse on-metal tags (foam spacer or specialized antenna design)
LiquidsRF absorption, reduced rangePosition tags away from liquid containers; use specialized tags
Dense tag stackingCoupling effects, missed readsIncrease session duration; use Session 1/2 for singulation
Multi-reader environmentsRF interference, ghost readsCoordinate reader operation; use frequency hopping
Extreme temperaturesTag IC sensitivity variesVerify tag operating temperature range in datasheet
