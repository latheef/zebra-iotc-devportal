alert_short

Compact alert format optimized for Mobile Device Management (MDM) systems. Provides essential alert information in a minimal payload for bandwidth-constrained environments.

Topic​

Topic
Event{tenantId}/MGMT/clients/event/{deviceSerial}

Note: alert_short events are published on the same MGMT event topic as full alerts. MDM systems can filter by the event field to process only compact alerts.

Direction​

Reader → Application

Payload​

{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747218125,
  "type": "BAT_LOW",
  "sev": "W",
  "val": 18,
  "thr": 20
}
{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747221030,
  "type": "BAT_CRIT",
  "sev": "C",
  "val": 4,
  "thr": 5
}
{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747214700,
  "type": "TEMP_HI",
  "sev": "W",
  "val": 52,
  "thr": 50
}
{
  "event": "alert_short",
  "sn": "24042AR50033",
  "ts": 1747210800,
  "type": "PWR_CHG",
  "sev": "I",
  "val": 1,
  "thr": 0
}
FieldTypeDescription
eventstringAlways "alert_short"
snstringDevice serial number (abbreviated field name)
tsintegerUnix timestamp (seconds since epoch)
typestringShort alert type code (see table below)
sevstringSeverity code: I (info), W (warning), C (critical)
valnumberCurrent value that triggered the alert
thrnumberConfigured threshold

Alert Type Codes​

CodeFull Alert TypeDescription
BAT_LOWBATTERY_LOWBattery below low threshold
BAT_CRITBATTERY_CRITICALBattery below critical threshold
TEMP_HITEMPERATURE_HIGHTemperature above threshold
NET_DCNETWORK_DISCONNECTNetwork interface disconnected
PWR_CHGPOWER_SOURCE_CHANGEPower source changed

Severity Codes​

CodeMeaning
IInfo
WWarning
CCritical

Notes​

  • alert_short uses abbreviated field names and Unix timestamps to minimize payload size, typically under 100 bytes per alert.
  • The val field semantics depend on the alert type: battery percentage for BAT_LOW/BAT_CRIT, temperature in Celsius for TEMP_HI, and a numeric indicator for PWR_CHG and NET_DC.
  • For PWR_CHG, val encodes the new power source: 0 = CRADLE, 1 = BATTERY, 2 = USB.
  • alert_short events are generated alongside full alerts events. Both are published when an alert condition is triggered.
  • MDM systems that do not need the full alerts payload can subscribe to the same topic and filter for "event": "alert_short".
