alert_short

Compact alert notification for lightweight event logging and real-time monitoring. Provides a short identifier, type, priority, and human-readable description for each alert condition.

Topic​

Topic
Event{tenantId}/MGMT/clients/event/{deviceSerial}

Direction​

Reader → Application

Payload​

Firmware Download Success​

{
  "id": "FIRMWARE_DOWNLOAD_SUCCESS",
  "timestamp": "22:54:25",
  "type": "ALERT",
  "priority": "LOW",
  "messageID": "ZEBRA_RFID_HH_ALERTS",
  "description": "firmware download completed successfully"
}

FieldTypeDescription
idstringAlert identifier (see full enum table below)
timestampstringTime when the alert was generated
typestringCategory: NOTIFICATION or ALERT
prioritystringUrgency level: CRITICAL, HIGH, or LOW
messageIDstringAlways "ZEBRA_RFID_HH_ALERTS"
descriptionstringHuman-readable description of the alert condition

Notes​

  • CRITICAL priority alerts (e.g., BATTERY_CRITICAL_SET) require immediate attention.
  • LOW priority alerts (e.g., FIRMWARE_DOWNLOAD_SUCCESS) are informational.
  • alert_short events are published on the same MGMT event topic as full alerts events. Filter by event content to distinguish them.
  • The messageID field is always "ZEBRA_RFID_HH_ALERTS" and can be used to identify this event type.
