Handheld Reader MQTT API Reference

Complete MQTT API reference for the Zebra IoT Connector on RFD40 / RFD90 Series handheld RFID readers. The API covers device management, RFID operation control, tag data collection, and fleet monitoring via MQTT 3.1.1.

Interactive Reference

Open the Interactive AsyncAPI Reference → for the full schema explorer with request/response payloads, enums, and field-level documentation.

Topic Pattern​

All topics follow this structure:

{tenantId}/{interface}/clients/{channel}/{deviceSerial}
SegmentDescription
tenantIdUnique organization ID assigned during provisioning
interfaceMGMT, CTRL, DATA, or MDM
channelcmnd, resp, event, data1event, or data2event
deviceSerialDevice serial number (e.g., RFD9030-12345)

Channels & Interfaces​

MGMT: Device Management (17 endpoints)​

ChannelDirectionDescription
{tenantId}/MGMT/clients/cmnd/{deviceSerial}PublishSend management commands
{tenantId}/MGMT/clients/resp/{deviceSerial}SubscribeReceive management responses
{tenantId}/MGMT/clients/event/{deviceSerial}SubscribeReceive device events

Commands:

CommandDescription
GET_STATUSGet device status including battery, memory, RFID state
GET_VERSIONGet firmware and hardware version information
GET_CURRENT_REGIONGet the current regulatory region
GET_CONFIGGet full device configuration
SET_CONFIGApply device configuration changes
SET_WIFIConfigure Wi-Fi network (WPA2/WPA3, Enterprise)
GET_WIFIGet current Wi-Fi configuration
DELETE_WIFI_PROFILERemove a saved Wi-Fi profile
GET_ETHGet Ethernet/cradle network configuration
CONFIG_ENDPOINTAdd, modify, or delete MQTT endpoints
GET_ENDPOINT_CONFIGGet current MQTT endpoint configuration
INSTALL_CERTIFICATEInstall TLS/mTLS certificates (Base64)
DELETE_CERTIFICATERemove an installed certificate
GET_INSTALLED_CERTIFICATEList installed certificates
CONFIG_EVENTSConfigure event thresholds (heartbeat, temperature, battery)
SET_OSInitiate firmware update from URL
REBOOTReboot the device

Events:

EventDescription
heartBeatEVTPeriodic device health (battery %, temperature, network)
alertsCritical alerts (low battery, high temp, storage)
alert_shortLightweight alert notifications
exceptionEVTError reports (RFID, network, config, certificate)
mqttConnEVTMQTT connection status changes

CTRL: RFID Operation Control (5 endpoints)​

ChannelDirectionDescription
{tenantId}/CTRL/clients/cmnd/{deviceSerial}PublishSend RFID control commands
{tenantId}/CTRL/clients/resp/{deviceSerial}SubscribeReceive control responses

Commands:

CommandDescription
SET_OPERATING_MODEConfigure RFID profiles, antennas, tag filters, read/write operations
GET_OPERATING_MODEGet current RFID operating configuration
CONTROL_OPERATIONStart or stop RFID/Scanner operations
SET_POST_FILTERSet tag data post-filters (prefix, suffix, regex, RSSI, whitelist/blacklist)
GET_POST_FILTERGet current post-filter configuration

DATA: Tag Data Streaming (2 endpoints)​

ChannelDirectionDescription
{tenantId}/DATA/clients/data1event/{deviceSerial}SubscribePrimary tag data stream
{tenantId}/DATA/clients/data2event/{deviceSerial}SubscribeSecondary tag data stream

Tag data fields include: EPC, TID, antenna ID, RSSI, phase, channel index, timestamp, read count, memory bank data, PC bits, CRC, XPC, and GPS coordinates.
