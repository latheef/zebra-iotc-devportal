# Glossary

Key terms used throughout the Zebra IOTC documentation.

| Term | Definition |
|------|-----------|
| **AsyncAPI** | An open specification for describing event-driven APIs, used to document MQTT interfaces. |
| **EPC** | Electronic Product Code. A unique identifier stored on an RFID tag, typically 96 bits. |
| **Gen2 / Gen2x** | Refers to the EPC UHF Gen2 (ISO 18000-63) air interface standard. Gen2x extends it with additional features. |
| **IoT Connector (IOTC)** | Zebra firmware running on the reader that translates MQTT messages into hardware operations. |
| **MQTT** | Message Queuing Telemetry Transport. A lightweight publish-subscribe protocol used for device communication. |
| **mTLS** | Mutual TLS. Both client and server present certificates to authenticate each other. |
| **OpenAPI** | An open specification for describing RESTful APIs, used to document REST interfaces. |
| **QoS** | Quality of Service. MQTT defines three levels: 0 (at most once), 1 (at least once), 2 (exactly once). |
| **REST** | Representational State Transfer. An architectural style for web APIs using HTTP methods. |
| **RFID** | Radio-Frequency Identification. A technology that uses radio waves to identify and track tags attached to objects. |
| **RSSI** | Received Signal Strength Indicator. A measurement of the power level of a received radio signal. |
| **TLS** | Transport Layer Security. A cryptographic protocol for securing network communications. |
| **UHF** | Ultra-High Frequency. The 860–960 MHz band used by EPC Gen2 RFID systems. |
